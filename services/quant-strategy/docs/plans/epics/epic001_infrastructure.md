# EPIC-001: 策略引擎基础设施

**版本**: v1.0
**状态**: 📅 规划中
**优先级**: P0 (基石)

## 📋 概述
构建量化策略服务的底层基础设施，为上层策略（长线配置、波段增强）提供统一的数据、计算和回测环境。

## 🎯 目标
- 建立统一的策略开发标准和接口。
- 提供高效的数据访问层，对接 `get-stockdata` 服务。
- 实现策略参数配置化管理。
- 集成基础回测框架，支持策略验证。

## 📚 User Stories

### Story 1.1: 策略服务脚手架与配置
- **目标**: 初始化 `quant-strategy` 服务结构，配置数据库连接、日志和异常处理。
- **任务**:
  - 建立 FastAPI 项目结构。
  - 配置 Nacos 注册与发现。
  - 设置 ClickHouse 和 Redis 连接池。
  - 实现统一的日志记录格式。

### Story 1.2: 数据适配层 (Data Adapter)
- **目标**: 封装 `get-stockdata` 的 API，提供对内的统一 Python 接口。
- **任务**:
  - 实现 `MarketDataProvider` 类，封装行情、K线、财务数据获取。
  - 实现数据缓存机制（Redis），减少跨服务调用延迟。
  - 提供数据清洗和格式化工具（DataFrame 标准化）。

### Story 1.3: 策略基类设计 (BaseStrategy)
- **目标**: 定义所有策略必须遵循的接口规范。
- **任务**:
  - 定义 `BaseStrategy` 抽象类，包含 `initialize`, `on_bar`, `on_tick`, `generate_signal` 等方法。
  - 定义标准化的 `Signal` 数据结构（包含标的、方向、强度、时间、逻辑）。
  - 实现策略注册机制（Registry Pattern）。

### Story 1.4: 基础回测引擎集成
- **目标**: 提供简单的向量化回测能力，用于快速验证策略逻辑。
- **任务**:
  - 集成 `backtrader` 或实现轻量级 `pandas` 向量化回测器。
  - 支持传入历史数据进行策略跑测。
  - 生成基础回测报告（收益率、最大回撤、夏普比率）。

### Story 1.5: 任务调度系统
- **目标**: 按需触发策略计算（每日收盘、每周五等）。
- **任务**:
  - 集成 `APScheduler`。
  - 配置定时任务：每日盘后数据同步、每周选股计算、盘中实时信号扫描。
