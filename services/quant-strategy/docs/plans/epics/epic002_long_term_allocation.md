# EPIC-002: 长线资产配置系统

**版本**: v1.0
**状态**: 📅 规划中
**优先级**: P1 (核心业务)
**依赖**: EPIC-001 (基础设施)

## 📋 概述
构建基于基本面分析的长线资产配置系统。该系统旨在筛选出财务健康、估值合理且具有长期增长潜力的优质标的，构建核心股票池 (`Core Pool`)。系统不追求短期波动收益，而是注重安全边际和复利增长。

## 🎯 目标
- 实现基于财务指标的自动筛选与排除机制（一票否决）。
- 建立多维度的企业质量打分模型（盈利、成长、现金流）。
- 实现估值分析，确定买入与卖出的安全边际。
- 自动化生成和维护"长线核心股票池"。

## 📚 User Stories

### Story 2.1: 风险否决过滤器 (Risk Veto Filter)
- **目标**: 快速剔除存在重大财务风险或合规风险的标的，确保"在此之上皆为安全"。
- **依赖**: Story 1.7 (Risk Control Module), Story 1.2 (Data Provider)
- **任务**:
  - 对接 `get-stockdata` 获取财务风控数据（商誉、质押率、流动比率等）。
  - 实现具体的 `RiskRule` 子类：
    - `GoodwillRiskRule`: 商誉占净资产比例过高拦截。
    - `PledgeRiskRule`: 大股东质押率过高拦截。
    - `CashflowRiskRule`: 经营性现金流持续为负拦截。
    - `FinancialFraudRule`: 典型的财务造假特征拦截（存贷双高）。
  - 集成到 `RiskManager`，作为进入股票池的第一道门槛。

### Story 2.2: 基本面评分引擎 (Fundamental Scoring)
- **目标**: 对通过风控过滤的标的进行多维度打分，量化企业质量。
- **任务**:
  - 设计评分模型接口 `FundamentalScorer`。
  - 实现三个维度的子评分模块：
    - **盈利能力 (Profitability)**: ROE, ROIC, 毛利率趋势。
    - **成长能力 (Growth)**: 营收增速, 净利润增速, PEG。
    - **运营效率 (Efficiency)**: 周转率, 存货周转天数。
  - 数据标准化处理（行业内排名/分位数）。
  - 输出综合评分 (0-100分)。

### Story 2.3: 估值分析器 (Valuation Analyzer)
- **目标**: 计算标的的内在价值与当前价格的偏离度，确定安全边际。
- **任务**:
  - 实现多种估值模型：
    - PE/PB Band: 历史百分位估值。
    - DCF (简化版): 自由现金流折现。
    - PEG: 成长与估值匹配度。
  - 定义 `ValuationStatus`: 严重低估 / 低估 / 合理 / 高估 / 泡沫。
  - 提供买入/卖出建议价格区间。

### Story 2.4: 核心股票池管理 (Core Pool Management)
- **目标**: 自动化维护长线优选股票池。
- **任务**:
  - 定时调度（每周/月）执行全市场扫描。
  - 流程编排: 全市场 -> 风险否决 -> 评分筛选(Top 10%) -> 估值过滤 -> 核心池。
  - 股票池持久化存储 (MySQL/ClickHouse)。
  - 变动通知 (调入/调出提醒)。
