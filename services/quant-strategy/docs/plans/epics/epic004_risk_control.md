# EPIC-004: 风险控制与交易管理

**版本**: v1.0
**状态**: 📅 规划中
**优先级**: P1 (关键保障)
**依赖**: EPIC-001

## 📋 概述
构建全系统的风控防线，并辅助策略信号的实盘执行。确保单股亏损有限，总账户回撤可控，并严格执行止盈止损。

## 🎯 目标
- 强制执行严格的风控规则（单股2%，总回撤15%）。
- 动态管理仓位，根据市场冷热调整风险暴露。
- 提供清晰的交易指令触达用户。

## 📚 User Stories

### Story 4.1: 全局风控规则引擎
- **目标**: 在信号生成阶段和持仓阶段进行风险阻断。
- **任务**:
  - 实现单股风控：检查拟买入金额造成的潜在最大亏损是否超过总资金 2%。
  - 实现账户风控：监控总净值回撤，若触及 15% 警戒线，强制限制开新仓或触发减仓建议。
  - 实现黑名单机制（ST股、即将退市、财报暴雷）。

### Story 4.2: 智能止盈止损监控
- **目标**: 自动化监控持仓盈亏，生成平仓信号。
- **任务**:
  - **止损逻辑**: 监控个股跌破 10日线 或 亏损 > 8%，触发“果断止损”信号。
  - **止盈逻辑**: 监控收益 > 10-15%，触发“分批止盈”信号；监控高位滞涨/放量下跌，触发“全部止盈”。

### Story 4.3: 动态仓位管理系统
- **目标**: 根据市场情绪调节整体仓位水位。
- **任务**:
  - 定义情绪指数（0-1）。
  - 实现仓位调整逻辑：情绪 > 0.8 (过热) -> 提示减仓；情绪 < 0.3 (冰点) -> 提示分批建仓。
  - 维护“现金缓冲”比例 (10-15%)。

### Story 4.4: 信号执行与通知服务
- **目标**: 将策略产生的买卖信号及时通知给用户。
- **任务**:
  - 设计 Signal Event 数据结构。
  - 实现消息推送（WebSocket / 邮件 / 钉钉 / 企业微信）。
  - 提供“交易日志”功能，记录所有信号触发原因和时间，辅助盘后复盘。
