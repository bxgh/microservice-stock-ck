## 4. 环境与配置问题排查

在验证过程中还发现了以下环境问题并已解决：

1.  **容器代理拦截**:
    - **现象**: 容器内访问 `127.0.0.1:8083` 失败或超时。
    - **原因**: `http_proxy` 环境变量导致请求被转发到外部代理服务器。
    - **解决**: 在 Python 脚本中使用 `aiohttp.ClientSession(trust_env=False)` 绕过代理，或使用 `curl --noproxy "*"`。

2.  **服务假死**:
    - **现象**: 端口 TCP 通但 HTTP 请求挂起。
    - **解决**: 重启 `get-stockdata` 服务后恢复。
