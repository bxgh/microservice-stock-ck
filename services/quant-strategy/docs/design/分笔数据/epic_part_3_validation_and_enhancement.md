### Story 004.01: 滚动窗口回测框架 (BacktestSimulator)
**优先级**: P0  
**来源**: `7_backtest.md`

#### 目标
构建能够回模拟历史资金团聚类并进行交易预测的仿真环境，验证策略的 Alpha 来源。

#### 核心逻辑

##### 1. Train/Test 滚动机制
采用 **Walk-Forward Analysis** (前向步进回测)：
- **训练窗口** $T_{train}$ (3-5天)：
  - 计算距离矩阵与聚类
  - 识别 Cluster 龙头与跟随者
  - 训练 OBI 动量阈值
- **验证窗口** $T_{test}$ (次日)：
  - 模拟 9:30-15:00 逐分钟交易
  - 记录信号生成与盈亏归因
- **滚动步长**：1 天

##### 2. 交易仿真细节
- **T+1 约束**：当日买入，次日可卖（需管理持仓队列）
- **滑点模型 (Slippage)**：
  $$Slippage = \lambda \times \sqrt{\frac{OrderSize}{AvgVolume}} \times Volatility$$
  - 基于 Kyle's Lambda 动态估算冲击
  - 小盘股滑点惩罚系数 1.5x
- **费率设置**：佣金 0.03% + 印花税 0.1% (卖出)

##### 3. 归因分析
输出盈亏来源分解：
- Cluster Beta 收益（板块红利）
- Leader-Follower Alpha 收益（择时能力）
- 隔夜跳空收益（事件驱动）

#### 验收标准
- [ ] 输出标准化绩效报告（Sharpe, Sortino, MaxDrawdown, WinRate, ProfitFactor）
- [ ] 完成参数敏感性分析热力图（DTW窗口 vs 聚类阈值）
- [ ] 验证 "龙头股" 的领先性（跟随者收益滞后相关性显著）

### Story 004.02: 工程鲁棒性与分布式加速 (EngineeringPlus)
**优先级**: P1  
**来源**: `6_engineering.md`

#### 目标
确保策略能够稳定地每日运行，并具备大规模计算任务的容错能力。

#### 核心逻辑

##### 1. 增量计算 (Incremental Computing)
- **指纹比对**：计算当日特征向量与前一日的 Cosine 相似度
- **惰性更新**：若相似度 > 0.98（行为模式未变），复用昨日计算的局部距离矩阵
- 预计减少 40% 重复计算量

##### 2. 存储优化
- **矩阵序列化**：使用 `PyArrow` / `Parquet` 存储稀疏距离矩阵，IO 速度提升 10x
- **内存映射**：对于超大矩阵（5000x5000），使用 `numpy.memmap` 避免 OOM

##### 3. 容错机制
- **断点续传**：记录每个 Batch（如100只股票）的计算进度到 Redis
- **优雅降级**：若某只股票数据缺失，自动从矩阵中剔除，不中断整体聚类

#### 验收标准
- [ ] 策略全流程在 2000 节点 Cluster 上耗时 < 45 分钟
- [ ] 内存峰值控制在 64GB 以内
- [ ] 模拟 ClickHouse 宕机场景，系统能从 checkpoint 恢复

### Story 004.03: 日内动量与隔夜套利 (IntradayEngine) [P1 增强]
**优先级**: P1  
**来源**: `A3_intraday_momentum.md`

#### 目标
利用盘后分析出的龙头关系，拦截日内动量爆发的机会。

#### 核心逻辑

##### 1. 隔夜跳空 (Gap) 策略
- **筛选对象**：昨日 Cluster 龙头股
- **信号触发**：
  - 高开 > 2% 且 9:30-9:35 成交量放大（VolumeRatio > 2.0）
  - 逻辑：强势延续，追涨
- **反转过滤**：
  - 若 VPIN > 0.6 或 OBI 动量 < 0，判定为诱多，放弃交易

##### 2. 动量传导预测
- **监控窗口**：9:30 - 10:00
- **触发逻辑**：
  - 当龙头涨幅 > 3% 且跟随者涨幅 < 1%
  - 且跟随者 OBI 动量转正（买单堆积）
  - 发出“补涨买入”信号

##### 3. 收盘效应 (End-of-Day)
- 利用开盘 30 分钟收益预测收盘 30 分钟趋势
- 若预测显著为正，且 Cluster 分歧度低，执行尾盘买入（博弈次日溢价）

#### 接口定义
```python
class IntradaySignal:
    timestamp: datetime
    cluster_id: int
    leader_stock: str
    target_stock: str
    signal_type: Enum("GAP_FOLLOW", "LAG_CATCHUP", "EOD_ALPHA")
    confidence: float
```

#### 验收标准
- [ ] 成功捕捉到历史数据中的“龙头涨停-跟随者补涨”形态
- [ ] 隔夜策略胜率 > 55%，盈亏比 > 1.2
- [ ] 日内信号延迟 < 300ms（从行情到达至信号生成）
