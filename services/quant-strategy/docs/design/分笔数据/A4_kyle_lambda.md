# Kyle's Lambda - 价格冲击系数

> **优先级**: P2 - 中优先
> **理论来源**: Albert Kyle (1985), UC Berkeley, NBER

## 1. 核心假设

**假设**: 价格变化对成交量的敏感度（λ）反映市场流动性和信息不对称程度。

**实证支持**:
- λ越高 → 市场深度越浅 → 微小失衡导致大幅波动
- 知情交易者活跃时λ显著上升
- λ可作为流动性风险的领先指标

---

## 2. 理论模型

### 2.1 Kyle (1985) 模型

Kyle模型是市场微观结构的基石，描述知情交易者、做市商、噪音交易者的博弈。

**核心关系**:

$$\Delta P = \lambda \cdot Q + \epsilon$$

其中：
- $\Delta P$：价格变化
- $Q$：带符号成交量（买正卖负）
- $\lambda$：价格冲击系数
- $\epsilon$：噪音

### 2.2 λ的经济含义

| λ值 | 市场状态 | 含义 |
|-----|----------|------|
| 低 | 流动性充足 | 大单冲击小 |
| 高 | 流动性紧张 | 小单即引发波动 |
| 突增 | 信息不对称加剧 | 知情交易活跃 |

---

## 3. 计算方法

### 3.1 回归估计

对分钟级数据进行回归：

$$\Delta P_t = \alpha + \lambda \cdot SignedVolume_t + \epsilon_t$$

**实现步骤**:
1. 计算每分钟价格变化：$\Delta P_t = P_{t}^{close} - P_{t-1}^{close}$
2. 计算带符号成交量：$SV_t = (BuyVolume_t - SellVolume_t)$
3. 线性回归求斜率λ

### 3.2 滚动窗口

为捕捉λ的动态变化，使用滚动窗口估计：

| 窗口类型 | 长度 | 用途 |
|----------|------|------|
| 短期 | 10分钟 | 实时监控 |
| 中期 | 30分钟 | 日内分析 |
| 长期 | 全天 | 跨日对比 |

---

## 4. 信号解读

### 4.1 λ绝对值信号

| 条件 | 市场状态 | 操作建议 |
|------|----------|----------|
| λ处于历史低位 | 流动性充沛 | 可加大仓位 |
| λ处于历史高位 | 流动性紧张 | 减仓或观望 |
| λ突然翻倍 | 信息冲击 | 警惕大波动 |

### 4.2 λ变化率信号

$$\Delta\lambda = \frac{\lambda_{current} - \lambda_{prev}}{\lambda_{prev}}$$

- Δλ > 50% → 流动性快速恶化
- Δλ < -30% → 流动性恢复

---

## 5. 与原框架集成

### 5.1 Cluster内部风险排序

对Cluster内股票按λ排序：
- λ低的股票：流动性好，优先操作
- λ高的股票：流动性差，慎重操作

### 5.2 信号执行过滤

在生成买入信号时：
```
if λ > threshold:
    reduce_order_size()  # 降低下单量
    use_limit_order()    # 使用限价单
```

### 5.3 与VPIN联合使用

| VPIN | λ | 状态 | 操作 |
|------|---|------|------|
| 低 | 低 | 健康 | 正常交易 |
| 高 | 低 | 知情交易但流动性足 | 谨慎 |
| 低 | 高 | 流动性差但无信息冲击 | 小仓位 |
| 高 | 高 | 危险 | 停止交易 |

## 6. Level-1数据适配

### 6.1 数据需求

| 字段 | 必需性 | Level-1可用性 |
|------|--------|---------------|
| 成交价 | ✅ 必需 | ✅ 可用 |
| 成交量 | ✅ 必需 | ✅ 可用 |
| 买卖方向 | ✅ 必需 | ⚠️ 需推断 |

### 6.2 方向推断

使用Lee-Ready规则简化版：
- 成交价 > 上一成交价 → 买入
- 成交价 < 上一成交价 → 卖出
- 成交价 = 上一成交价 → 按Tick规则（由上一方向继承）

---

## 7. 回测验证要点

### 7.1 验证假设

1. λ高位时，后续价格波动是否更大？
2. λ突增后，胜率是否下降？
3. 低λ股票的信号执行滑点是否更小？

### 7.2 对比实验

| 策略 | 描述 | 预期 |
|------|------|------|
| 无过滤 | 忽略λ | 基准 |
| λ过滤 | λ过高时不交易 | 降低滑点 |
| λ加权 | 低λ时增加仓位 | 提高收益 |

## 8. 参数配置

```yaml
kyle_lambda:
  # 滚动窗口
  rolling:
    short_window: 10   # 分钟
    medium_window: 30
    long_window: 240
    
  # 阈值（需基于历史分位数确定）
  thresholds:
    high_percentile: 80  # λ超过80%分位数视为高
    low_percentile: 20
    
  # 变化率阈值
  change_rate:
    surge: 0.5   # 50%突增
    recover: -0.3
```
