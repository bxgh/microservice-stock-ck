# Antigravity 全流程代码开发指南

## 核心能力概述

Antigravity 支持**自动化**和**半自动化**完成从需求到部署的完整软件开发流程，包括产品设计、代码编写、质量检查、测试和文档生成。

---

## 自动化模式

### 完全自动模式
- 用户提供需求描述
- AI 独立完成所有开发阶段
- 仅在关键决策点请求确认
- 适用：小型项目、原型开发、学习场景

### 半自动模式（推荐）
- 每个阶段完成后暂停
- 生成方案供用户审核
- 用户批准后执行下一阶段
- 适用：中大型项目、生产环境

### 纯辅助模式
- 用户主导编码
- AI 提供建议和代码片段
- 适用：学习新技术、代码审查

## 提示词使用方法

### 一句话模式（最简）
**结构**：需求描述 + 技术栈

**必要信息**：
- 项目功能描述
- 技术栈要求

**可选信息**：
- 特殊需求
- 约束条件

---

### 分步详细模式（推荐）
**每阶段必须包含**：
- 当前阶段目标
- 具体要求
- 验收标准

**结构**：
```
阶段名称：
目标：
要求：
- 技术选型
- 功能清单
- 质量标准
- 特殊约束
输出：期望的交付物
```

---

### 质量检测专用提示词
**必要信息**：
- 代码路径
- 检查范围

**可选信息**：
- 检查重点（安全/性能/规范）
- 自动修复权限
- 报告格式要求

## 模型切换方法

### 方法 1: 全局切换
**操作**：界面右上角 Model Selection

**影响范围**：所有后续对话

**常用模型选择**：
- **Claude 4.5 Sonnet**: 代码审查、复杂逻辑、大代码库
- **GPT-4o**: 快速生成、多语言、原型开发
- **Gemini 2.5 Pro**: 大上下文分析、文档处理
- **o1**: 复杂算法、数学问题、深度推理

---

### 方法 2: 对话中指定
**语法**：在提示词中明确说明使用哪个模型

**格式**：
```
用 [模型名] 完成 [任务]
```

**影响范围**：仅指定的任务

---

### 方法 3: 自动选择（推荐）
AI 根据任务类型自动选择最优模型

**自动选择逻辑**：
- 需求分析/文档阅读 → Claude（长上下文）
- 快速原型/代码生成 → GPT-4o（速度快）
- 代码审查/调试 → Claude（细致）
- 算法优化 → o1（推理强）
- 测试生成 → GPT-4o（并行能力）

## 工作流程模板

### 完整项目开发流程
```
1. 提供需求 → AI 生成技术方案
2. 审核方案 → 用户确认或修改
3. 自动编码 → AI 创建所有文件
4. 质量检查 → 自动运行 Linter/测试
5. 审核报告 → 用户批准修复方案
6. 自动修复 → AI 优化代码
7. 生成文档 → README/API 文档/部署指南
8. 交付验收 → 用户运行和测试
```

---

### 代码质量检测流程
```
1. 指定代码路径
2. 定义检查范围（规范/安全/性能）
3. AI 执行检测
4. 生成报告（问题清单 + 修复建议）
5. 用户确认修复方案
6. AI 自动修复（可选）
7. 重新验证
8. 生成最终报告
```

---

### 代码重构流程
```
1. 指定重构目标（性能/可读性/架构）
2. AI 分析现有代码
3. 生成重构方案
4. 用户审核方案
5. AI 执行重构
6. 运行测试验证
7. 生成变更对比报告
```

## 提示词质量标准

### 高质量提示词特征
- **明确性**: 清晰的目标和边界
- **完整性**: 包含必要的技术细节
- **可验证性**: 定义明确的验收标准
- **优先级**: 区分必需和可选功能

### 低质量提示词特征（避免）
- 模糊的需求描述
- 缺少技术约束
- 无明确的交付标准
- 过度宽泛的范围

## 控制粒度选择

| 控制级别 | 用户输入次数 | 决策权 | 开发速度 | 适用场景 |
|---------|-----------|--------|---------|---------|
| 粗粒度 | 1-2次 | AI主导 | 极快 | 原型/学习 |
| 中粒度 | 3-5次 | 共同决策 | 快 | 常规项目 |
| 细粒度 | >10次 | 用户主导 | 慢 | 关键系统 |

## 限制与约束

### AI 能做的
- 技术实现自动化（99%）
- 代码质量检查（100%）
- 测试生成（90%）
- 文档生成（100%）

### AI 不能做的
- 业务决策（需求是否合理）
- 用户体验判断（界面好不好看）
- 产品规划（做什么功能）
- 实际部署执行（只能生成脚本）

### 需要人工确认的
- 重大架构决策
- 安全敏感操作
- 生产环境修改
- 破坏性变更
