## 服务清单

| 服务 | 来源 | 运行模式 | 仓库 | 职责 |
|:-----|:-----|:---------|:-----|:-----|
| **gsd-api** | 从 get-stockdata 拆出 | 长驻 | services/gsd-api/ | 数据查询 |
| **gsd-sync** | 从 get-stockdata 拆出 | 临时 | services/gsd-sync/ | MySQL→CH 同步 |
| **gsd-quality** | 从 get-stockdata 拆出 | 临时 | services/gsd-quality/ | 质量检测+修复 |
| **gsd-shared** | 新建 | 包 | libs/gsd-shared/ | 共享模型 |
| **task-orchestrator** | 升级 task-scheduler | 长驻 | services/task-orchestrator/ | 任务编排 |

## 每日任务流程

```
18:00  task-orchestrator 检测: 今天是交易日？
       │
       ├─ 是 ──▶ 启动 gsd-sync 容器 (4 并行分片)
       │              │
       │              ├── shard-0: 股票 000001-001500
       │              ├── shard-1: 股票 001501-003000
       │              ├── shard-2: 股票 003001-004500
       │              └── shard-3: 股票 004501-XXXXXX
       │              │
       │              └── 全部完成 (~2.5分钟)
       │
       ├──▶ 销毁 sync 容器
       │
       ├──▶ 启动 gsd-quality 容器
       │         │
       │         ├── 完整性检测
       │         ├── 缺失 < 50 只？自动修复
       │         └── 完成
       │
       ├──▶ 销毁 quality 容器
       │
       └──▶ 发送通知: 今日数据同步完成
```

## 技术决策

| 决策点 | 选择 | 理由 |
|:-------|:-----|:-----|
| 服务拆分 | 完全独立仓库 | 第一性原理正确 |
| 同步服务运行模式 | 临时任务 | 资源最优，可并行 |
| 质量服务运行模式 | 临时任务 | 资源最优 |
| 并行策略 | 分片 (4 容器) | 10分钟→2.5分钟 |
| 共享代码 | gsd-shared 包 | 模型一致性 |
