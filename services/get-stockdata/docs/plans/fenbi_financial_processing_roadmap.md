# 分笔数据(Tick)深度处理与金融增强路线图

## 1. 思维转变：从"数据清洗"到"数据挖掘"

当前的 `fenbi_engine` 主要关注数据的**工程质量**（格式统一、去重、排序），这是必要的基础。作为金融专家，建议将关注点提升到数据的**金融价值**。Tick 数据不仅是价格的记录，更是市场参与者博弈的痕迹。

处理流程应从 `Raw Data -> Clean Data` 升级为 `Raw Data -> Clean Data -> Enriched Data -> Alpha Features`。

---

## 2. 核心增强模块建议 (Financial Enhancement Modules)

### 2.1 交易博弈分析 (Trade Flow Analysis)

Tick 数据中最核心的信息是**“谁在买？谁在卖？力度如何？”**。

*   **主动买卖推导 (Trade Direction Inference)**
    *   **现状**：原始数据可能不包含“主动买入/卖出”标识，或者标识不准确。
    *   **专家建议**：引入 **Lee-Ready 算法** 或 **价格升降法**。
        *   如果 成交价 > (买一价 + 卖一价)/2，推导为主动买入。
        *   如果 成交价 < (买一价 + 卖一价)/2，推导为主动卖出。
    *   **价值**：这是计算“资金流向”的基础，直接反映市场情绪。

*   **大单监控与拆解 (Large Order Tracking)**
    *   **概念**：机构的大额交易往往会拆分成多个小单（Iceberg Orders）执行。
    *   **建议**：
        *   **阈值监控**：标记超过一定金额（如100万）或手数的单笔成交。
        *   **高频连续单识别**：识别短时间内连续发生的、方向相同的中型单，这往往是算法交易（TWAP/VWAP）的痕迹。

### 2.2 市场微观结构特征 (Market Microstructure Features)

利用高频数据特有的属性构建预测因子。

*   **订单流不平衡 (Order Flow Imbalance, OFI)**
    *   **定义**：单位时间内的（主动买入量 - 主动卖出量）。
    *   **价值**：微观结构理论中，OFI 是短期价格变动最强的解释变量。

*   **VWAP (成交量加权均价)**
    *   **定义**：$\sum(Price * Volume) / \sum(Volume)$
    *   **价值**：机构交易员的业绩基准。当前价格与当日 VWAP 的偏离度，是判断价格“贵贱”的重要锚点。

*   **已实现波动率 (Realized Volatility)**
    *   **定义**：基于高频收益率计算的波动率（如每分钟收益率平方和）。
    *   **价值**：比日线级别的 ATR 或标准差更灵敏，能捕捉盘中的恐慌情绪。

### 2.3 时序重采样 (Time-Series Resampling)

Tick 数据量大且噪点多，直接用于分析往往效果不佳。需要将其“降维”。

*   **Tick to OHLCV (K线生成)**
    *   **功能**：将 Tick 流实时聚合成 1分钟、5分钟 Bar。
    *   **建议**：不仅支持**时间等分**（Time Bars），更应支持**非等时采样**：
        *   **Volume Bars**：每成交 1000 手生成一根 K 线。
        *   **Dollar Bars**：每成交 1000 万元生成一根 K 线。
    *   **专家视角**：Volume/Dollar Bars 的统计特性（正态性）优于 Time Bars，更适合数学模型和机器学习。

## 3. 存储与架构优化 (Infrastructure Upgrade)

### 3.1 存储格式升级
*   **现状**：CSV/Excel。
*   **问题**：读写慢，丢失类型信息，体积大。
*   **建议**：引入 **Parquet** 或 **Feather** 格式。
    *   **列式存储**：极快的数据读取速度（适合回测）。
    *   **压缩**：体积通常是 CSV 的 1/10。
    *   **类型安全**：原生支持时间戳和浮点数，无需反复解析。

### 3.2 计算架构
*   **向量化计算**：继续保持并强化 NumPy/Pandas 的向量化操作，避免 Python 循环。
*   **流式计算 (Streaming)**：对于实时性要求高的场景（如盘中监控），架构应支持逐笔处理（Event-driven），而不仅是盘后批量处理（Batch）。

---

## 4. 实施优先级建议

基于“价值最大化”原则，建议的实施顺序：

1.  **Phase 1: 金融语义清洗**
    *   实现 `TradeDirectionInference`（主动买卖推导）。
    *   实现 `MarketStateFilter`（区分集合竞价与连续竞价）。

2.  **Phase 2: 核心指标计算**
    *   实现 `VWAP` 和 `NetMoneyFlow`（净资金流向）。
    *   升级存储格式为 Parquet。

3.  **Phase 3: 高级特征与重采样**
    *   实现 `TickResampler` (Tick -> 1min Bar)。
    *   实现 `OrderFlowImbalance`。

---

## 5. 总结

从金融专家视角看，`fenbi_engine` 不应止步于“整理数据”，而应成为**“市场显微镜”**。通过引入主动买卖推导、微观结构因子和高效存储，该系统将具备支持专业量化研究和实盘监控的能力。
