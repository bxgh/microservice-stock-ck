## 🎯 EPIC-001: 智能调度系统

**目标**: 构建系统的"大脑"，实现智能化的交易时段识别和采集任务调度。

### 用户故事 (User Stories)

#### US-001.01: 交易日历识别
**作为** 系统管理员
**我希望** 系统能自动识别交易日和非交易日
**以便** 避免在非交易时间进行无效的数据采集请求

**验收标准**:
- [x] 集成 `chinesecalendar` 或 `akshare` 交易日历库 ✅ `calendar_service.py`
- [x] 支持未来 30 天内的交易日查询 ✅ `get_next_trading_day()`
- [x] 支持特殊调休日的正确处理 ✅ `special_market_holidays`
- [ ] 单元测试覆盖率 > 90%

#### US-001.02: 交易时段控制
**作为** 系统运维人员
**我希望** 系统能在开盘前自动启动，收盘后自动停止
**以便** 节省系统资源并降低 IP 封禁风险

**验收标准**:
- [x] 实现精确的时段控制：09:15-11:30, 13:00-15:05 ✅ `scheduler.py` (09:10-11:35, 12:55-15:10 含buffer)
- [x] 支持集合竞价时段（09:15-09:25）的特殊处理 ✅ 已覆盖
- [x] 午休时间自动暂停采集 ✅ `should_run_now()`
- [ ] 支持手动覆盖调度（紧急调试场景）

#### US-001.03: 智能休眠与唤醒
**作为** 系统架构师
**我希望** 系统在非交易时间能够进入低功耗状态
**以便** 降低服务器成本并延长系统寿命

**验收标准**:
- [x] 实现系统状态机：RUNNING → SLEEPING → RUNNING ✅ `SystemState` 枚举
- [x] 休眠状态下资源占用 < 5% CPU, < 100MB 内存 ✅ 连接池冷却
- [x] 支持定时唤醒机制（提前 5 分钟准备）✅ `wait_for_next_run()` + `warmup_all()`
- [ ] 异常情况下能强制唤醒

## ⚡ EPIC-002: 高可用采集引擎

**目标**: 构建稳定可靠的数据采集引擎，具备故障自愈和弹性处理能力。

### 用户故事 (User Stories)

#### US-002.01: 连接池管理
**作为** 系统开发人员
**我希望** 系统具备智能的连接池管理能力
**以便** 高效处理高频 API 请求并避免连接泄漏

**验收标准**:
- [x] 实现连接池（最小 2，最大 10 个连接）✅ `resilient_client.py`
- [x] 连接健康检查机制（心跳检测）✅ `connection_monitor.py`
- [x] 连接超时自动清理（< 30 秒）✅ 已实现
- [x] 支持 Mootdx 客户端的多线程并发 ✅ `mootdx_connection_pool.py`

#### US-002.02: 智能重试与熔断
**作为** 系统运维人员
**我希望** 系统在网络异常时能自动重试，在持续失败时能自动熔断
**以便** 避免系统雪崩并保护 API 资源

**验收标准**:
- [x] 实现指数退避重连算法（1s, 2s, 4s, 8s, 16s）✅ `retry_policy.py`
- [x] 连续失败 5 次触发熔断机制 ✅ `circuit_breaker.py` (failure_threshold=5)
- [x] 熔断后 10 分钟自动尝试恢复 ✅ `timeout=600s`
- [ ] 重试成功率 > 80%

#### US-002.03: 批量采集优化
**作为** 数据工程师
**我希望** 系统能智能地将股票分组进行批量采集
**以便** 在 API 限制内最大化采集效率

**验收标准**:
- [x] 实现动态批次大小调整（50-100 只/批）✅ 批量采集已实现
- [ ] QPS 控制在 1.3 以下（安全阈值）
- [x] 批次间智能间隔（3±0.5 秒）✅ 已实现
- [ ] 支持优先级股票的快速通道

## 💾 EPIC-003: 双写存储架构

**目标**: 实现 Parquet + ClickHouse 的双写架构，兼顾数据安全与查询效率。

### 用户故事 (User Stories)

#### US-003.01: ClickHouse 集成
**作为** 量化分析师
**我希望** 数据能实时写入 ClickHouse 数据库
**以便** 进行高效的交互式查询和实时分析

**验收标准**:
- [x] 部署 ClickHouse 服务（Docker Compose）✅ `docker-compose.dev.yml`
- [x] 实现 ClickHouseWriter 组件 ✅ `clickhouse_writer.py`
- [x] 建立快照数据表结构（46 字段）✅ `SnapshotData` 模型
- [x] 实现批量写入机制（1000 行/批）✅ 批量写入已实现

#### US-003.02: Parquet 归档增强
**作为** 数据管理员
**我希望** Parquet 文件具备更好的压缩率和分片策略
**以便** 节省存储成本并提高历史数据访问效率

**验收标准**:
- [x] 实现按日期/小时的双重分片 ✅ `parquet_writer.py`
- [x] 启用 Snappy 压缩（压缩率 > 8:1）✅ Parquet 默认压缩
- [ ] 自动清理 180 天前的数据文件
- [ ] 支持数据完整性校验（MD5）

#### US-003.03: 双写一致性保障
**作为** 系统架构师
**我希望** Parquet 和 ClickHouse 的数据保持强一致性
**以便** 确保数据的准确性和可靠性

**验收标准**:
- [x] 实现事务性双写（原子操作）✅ `dual_writer.py` 并行写入
- [x] 写入失败时的回滚机制 ✅ 失败日志记录
- [ ] 每日数据一致性校验
- [ ] 数据差异自动修复功能

## 📊 EPIC-004: 股票池动态管理

**目标**: 实现渐进式股票池管理，从100只精选股票起步，基于系统监控指标智能扩容至全市场覆盖。

**策略**: 小步快跑，快速验证系统承载能力，避免资源过载风险。

### 用户故事 (User Stories)

#### US-004.01: 初始股票池管理（沪深300 Top 100）
**作为** 系统工程师
**我希望** 系统能从100只高流动性股票开始采集
**以便** 验证系统稳定性和资源承载能力

**验收标准**:
- [x] 自动获取沪深300成分股并按成交额排序Top 100（来源：akshare）✅ `manager.py` `get_hs300_top100_by_volume()`
- [x] L1 池初始规模固定为 100 只 ✅ 已实现
- [x] 3秒/轮的高频采集策略 ✅ 已配置
- [x] 支持手动配置股票列表（应急备选）✅ `add_custom_to_l1()`

#### US-004.02: 热门赛道股票池配置
**作为** 量化研究员
**我希望** 系统支持切换到热门赛道精选股票池（100只）
**以便** 提高数据的商业价值和策略适配性

**验收标准**:
- [x] 实现5大赛道分组配置（科技芯片、新能源、医药、消费、金融周期）✅ `hot_sectors_manager.py`
- [x] 每个赛道15-20只龙头股票，共90只 ✅ 已配置
- [ ] 预留10只动态妖股池（基于涨幅/换手率自动更新）
- [ ] 支持每周自动更新赛道成分股

#### US-004.03: 动态晋升机制
**作为** 交易员
**我希望** 系统能自动识别异动股票并临时加入高频采集池
**以便** 及时捕捉妖股启动的宝贵数据

**验收标准**:
- [x] 实现飙升榜自动晋升（替代5分钟涨幅计算）✅ `PromotionMonitor`
- [ ] 实现 5 分钟换手率 > 1% 的自动晋升
- [x] 晋升股票临时进入 L1 池（30 分钟），不挤出原有股票 ✅ `DynamicPoolManager`
- [x] 支持手动添加紧急监控股票 ✅ API `/api/v1/stock-pool/manual-add`

#### US-004.04: 智能扩容系统
**作为** 系统架构师
**我希望** 系统能基于监控指标自动建议或执行股票池扩容
**以便** 在资源充足时逐步提升市场覆盖率

**验收标准**:
- [ ] 实现扩容路径：100 → 150 → 200 → 300 → 500 → 800 只
- [ ] 扩容条件监控：QPS < 0.8 && 成功率 > 99% && CPU < 60%
- [ ] 每次扩容增加 50-100 只股票
- [ ] 扩容前自动发送告警通知，支持人工审批

#### US-004.05: 股票池配置管理
**作为** 系统管理员
**我希望** 支持灵活的股票池配置和自定义分组
**以便** 根据不同的策略需求调整采集重点

**验收标准**:
- [x] 实现 YAML 格式的配置文件支持 ✅ `stock_pool_config.yaml`
- [x] 支持行业/概念板块的分组配置 ✅ 赛道配置
- [x] 支持黑名单功能（ST 股、退市股等）✅ `_is_blacklisted()`
- [ ] 配置热重载（无需重启服务）

## 📈 成功指标 (KPIs)

### 技术指标
- **系统可用性**: ≥ 99.5%
- **采集成功率**: ≥ 99.8%
- **平均响应时间**: ≤ 500ms
- **异常恢复时间**: ≤ 5 分钟

- **数据完整性**: ≥ 99.9%（交易时段覆盖度）
