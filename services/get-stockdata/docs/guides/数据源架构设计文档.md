# 数据源架构设计文档

## 架构概述

基于fenbi.py的成功数据获取逻辑，构建数据源抽象层，支持多数据源扩展。

## 核心组件

### 1. 数据源基类 (`src/data_sources/base.py`)

定义统一的数据源接口：
- `connect()` - 连接数据源
- `get_tick_data()` - 获取分笔数据
- `get_status()` - 获取连接状态
- `close()` - 关闭连接

### 2. 数据源工厂 (`src/data_sources/factory.py`)

- `create_source()` - 创建指定数据源实例
- `create_default_source()` - 创建默认数据源
- `get_available_sources()` - 获取可用数据源列表
- 支持配置管理和参数传递

### 3. mootdx实现 (`src/data_sources/mootdx/`)

#### 连接管理 (`connection.py`)
- 提取fenbi.py的连接逻辑
- 服务器选择和连接管理
- 最佳IP选择策略

#### 数据获取器 (`fetcher.py`)
- 保留fenbi.py验证过的多起始位置策略
- 异步处理和线程池管理
- 数据去重和质量控制

### 4. 业务引擎 (`src/services/fenbi_engine.py`)

- 包装数据源抽象层
- 保持原fenbi.py CLI接口
- 统计信息和错误处理
- 支持文件导出

## 设计原则

1. **向后兼容** - 保持原fenbi.py功能不变
2. **可扩展性** - 轻松添加新数据源
3. **模块化** - 清晰的组件分离
4. **配置驱动** - 统一的配置管理

## 使用示例

```python
# 创建引擎
engine = FenbiEngine('mootdx')

# 获取数据
data = await engine.get_tick_data('000001', '20241118')

# 导出文件
export_to_csv(data, 'output.csv')
```

## 扩展新数据源

1. 继承`DataSourceBase`基类
2. 实现必需的抽象方法
3. 在工厂类中注册
4. 添加配置信息
