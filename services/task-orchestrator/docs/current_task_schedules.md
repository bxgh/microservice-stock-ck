# Task Orchestrator 任务调度清单

本文档列出了当前系统配置的所有定时任务及其调度时间。

## 1. 核心运行任务 (已启用)

| 任务 ID | 任务名称 | 调度规律 (Cron/TradingCron) | 预计执行时间 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| `pre_market_gate` | **盘前数据质量门禁** | `0 8 * * 1-5` (交易日) | 每个交易日 **08:00** | 关键校验点，数据质量红线 |
| `daily_stock_collection` | **每日股票代码采集** | `5 9 * * *` | 每日 **09:05** | 拉取全市场股票清单 |
| `tick_data_migrate` | **分笔数据归档** | `0 9 * * 1-5` (交易日) | 每个交易日 **09:00** | 数据整理与空间清理 |
| `daily_cache_warmup` | **缓存预热** | `0 9 * * 1-5` (交易日) | 每个交易日 **09:00** | 提升 API 访问速度 |
| `daily_tick_sync_shard_0` | **分笔采集(Shard-0)** | `35 15 * * 1-5` (交易日) | 每个交易日 **15:35** | 盘后高频数据补采 |
| `daily_kline_sync` | **K线每日同步** | `30 17 * * *` (交易日) | 每个交易日 **17:30** | 分片同步行情数据 |
| `daily_strategy_scan` | **每日策略扫描** | `30 18 * * 1-5` (交易日) | 每个交易日 **18:30** | 计算选股信号 |
| `daily_db_backup` | **数据库备份** | `0 3 * * *` | 每日 **03:00** | 核心数据异地备份 |
| `weekly_log_cleanup` | **日志清理** | `0 2 * * 0` (周日) | 每周日 **02:00** | 清理 30 天前的旧日志 |
| `weekly_clickhouse_log_cleanup` | **ClickHouse日志清理** | `0 3 * * 0` (周日) | 每周日 **03:00** | 清理数据库系统日志 |

## 2. 实时采集服务 (常驻型)

这些任务作为容器服务常驻运行，内部自带市场时钟逻辑，仅在交易时段活跃。

| 服务 ID | 服务名称 | 运行时间 | 说明 |
| :--- | :--- | :--- | :--- |
| `intraday-tick-collector` | **盘中分笔实时采集** | 交易日 `09:25-15:00` | HS300 成分股实时分笔抓取与写入 |
| `snapshot-recorder` | **盘中快照录制器** | 交易日 `09:30-15:00` | L1 核心股票高频快照录制，双写存储 |

---

## 3. 补采与修复任务 (按需触发)

这些任务通常由 `CommandPoller` 监听云端指令执行，不执行自动定时调度。

| 任务 ID | 任务名称 | 触发方式 | 说明 |
| :--- | :--- | :--- | :--- |
| `repair_kline` | **K线数据补采** | 云端 API / SQL | 用于修复特定日期的 K 线缺失 |
| `repair_tick` | **分笔数据补采** | 云端 API / SQL | 用于修复特定日期的分笔缺失 |
