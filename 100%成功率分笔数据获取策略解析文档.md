## 🎯 关键技术要点

### 1. Start Position策略原理

**数据存储规律：**
- TongDaXin数据库按时间倒序存储分笔数据
- 最新交易记录存储在较小位置(0-1000)
- 历史数据存储在较大位置(4000+)
- **关键发现**：集合竞价数据(09:25)通常存储在4000位置附近

**位置时间映射：**
```
位置0-500：    最新100-200条记录 (14:50-15:00)
位置500-1500：  近期数据 (13:30-14:50)
位置1500-3000：  上午数据 (10:00-13:30)
位置3000-4500：  开盘数据 (09:30-10:00)
位置4000-5000：  集合竞价 (09:25-09:30) ← 核心目标区域
```

### 2. Offset策略原理

**批次大小设计：**
- **小offset(400-500)**：精确定位特定时间段
- **中offset(800-1200)**：覆盖较长时间段
- **大offset(1500-2000)**：确保数据完整性

**动态调整逻辑：**
- 早期使用较小offset进行精细搜索
- 后期使用较大offset确保覆盖完整
- 关键位置使用大offset确保获取09:25数据

### 3. 时间覆盖质量评估

```python
评估标准：
完美级别：≤09:25 (包含集合竞价)  ← 100%策略目标
```
