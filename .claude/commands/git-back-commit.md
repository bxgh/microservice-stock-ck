# Backend Smart Commit - 后端智能提交

## 命令描述

基于Git diff快速扫描和**分析**后端服务代码变更，**仅分析services/和packages/目录**，避免前端代码干扰，实现秒级精准**自动提交**。

> ✅ **重要提示**: 此命令执行完整的**分析、验证和自动提交**流程，包括代码质量检查、智能生成提交信息并执行git commit。

## 执行流程

### 1. 后端域扫描
- 执行 `git status --porcelain` 和 `git diff --name-only` 筛选后端变更文件
- **筛选规则**: 仅处理 `services/*` 和 `packages/*` 文件
- 跳过 `apps/*` 前端文件变更和 `docs/` 目录
- 快速分析变更类型：服务逻辑/共享包/基础设施

### 2. 后端变更分析
- **Python语法检查**: 仅检查Python文件的语法错误
- **API接口检查**: 验证API端点变更的兼容性
- **依赖分析**: 检查requirements.txt和包依赖变更
- **服务配置**: 检查Docker和配置文件变更
- **暂存状态检查**: 分析已暂存和未暂存的变更

### 3. 微服务关联检查
- **服务依赖**: 分析服务间的API调用关系
- **数据流检查**: 验证数据库schema变更影响
- **配置同步**: 检查跨服务配置一致性
- **Docker编排**: 验证docker-compose服务定义

### 4. 快速验证
- **Python语法**: `python -m py_compile` 快速验证
- **导入检查**: 验证Python import语句有效性
- **配置解析**: 验证YAML/JSON配置文件格式
- **Docker语法**: 检查Dockerfile和compose文件

### 5. 智能分析和准备
- **后域分析**: 识别services/和packages/变更
- **服务分类**: 按微服务分类生成提交信息
- **影响标注**: 标注受影响的其他服务
- **提交准备**: 自动暂存相关文件（可选）
- **部署提示**: 提供部署相关建议

## 后端域扫描规则

### 变更文件筛选
```bash
# 筛选后端相关文件（包括暂存和未暂存）
git status --porcelain | grep -E "^(A|M|\?\?).*services/"
git status --porcelain | grep -E "^(A|M|\?\?).*packages/"

# 或者仅分析已暂存的变更
git diff --cached --name-only | grep -E "^(services/|packages/)"
```

### 服务类型识别
- **数据服务**: data-collector, data-processor, data-storage
- **业务服务**: task-scheduler, notification, monitor
- **网关服务**: api-gateway, stock-data
- **共享包**: shared, utils, types, config

### 变更优先级
1. **核心服务变更**: task-scheduler, api-gateway
2. **数据服务变更**: data-* 系列服务
3. **辅助服务变更**: monitor, notification
4. **基础设施变更**: packages/config, scripts

## 注意事项

### ✅ 重要提醒
- **自动提交**: 此命令执行完整的**分析、验证和自动提交**流程
- **安全机制**: 只有通过所有验证检查后才会执行提交
- **暂存状态**: 命令会智能暂存后端相关变更文件
- **文档忽略**: 自动忽略docs/目录的变更（符合.gitignore规则）
- **回滚支持**: 如有问题可使用 `git reset HEAD~1` 回滚最近一次提交

### 🔧 技术注意事项
- **前端变更忽略**: 自动忽略apps/目录的所有变更
- **服务依赖**: 注意微服务间的依赖关系变更
- **配置变更**: Docker和配置文件变更需要特别关注
- **数据库变更**: schema变更需要提供迁移脚本

## 使用建议

### ✅ 推荐使用场景
- **后端服务开发**: Python服务逻辑更新
- **API接口修改**: REST API变更和兼容性
- **微服务重构**: 服务内部架构优化
- **基础设施更新**: Docker和配置调整
- **批量变更分析**: 多个服务的联合提交前检查

### ❌ 不推荐使用场景
- **全栈变更**: 同时涉及前后端的变更
- **数据库迁移**: 复杂的数据库结构变更
- **跨域重构**: 影响多个域的架构变更
- **紧急修复**: 需要立即提交的紧急变更

### 🚀 最佳实践
1. **开发阶段**: 完成代码修改后使用此命令分析
2. **提交前检查**: 在实际提交前验证代码质量
3. **团队协作**: 生成标准化的提交信息
4. **CI/CD集成**: 作为提交前的质量检查步骤

这个命令专注于后端微服务的智能分析和自动提交，提供一键式的代码质量检查和标准化提交流程，大幅提升开发效率。

## 🔒 安全机制

### 提交前验证检查
1. **语法检查**: 确保所有Python文件语法正确
2. **导入验证**: 检查Python import语句有效性
3. **配置解析**: 验证YAML/JSON配置文件格式
4. **依赖冲突**: 检查requirements.txt和包依赖
5. **API兼容**: 验证API端点变更的兼容性

### 失败处理
- **验证失败**: 不会执行提交，显示详细错误信息
- **部分失败**: 跳过有问题的文件，提交其他通过验证的变更
- **网络问题**: 使用本地缓存继续工作，网络恢复后同步
