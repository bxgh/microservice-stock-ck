# Smart Incremental Commit - 智能增量代码提交

## 命令描述

基于Git diff智能分析和提交前端代码变更，只检查实际修改的文件，实现精准高效的质量保证流程。

## 执行流程

### 1. 识别变更范围 (Git Diff分析)
- 执行 `git diff --name-only` 识别变更文件
- 分析变更类型：新增/修改/删除文件
- 识别变更影响范围：组件/页面/API/配置等
- 检查文件变更量和复杂度

### 2. 静态增量代码检查
- **语法检查**: 只检查变更文件的TypeScript语法
- **规范检查**: 验证变更代码的ESLint规则
- **类型检查**: 分析变更代码的类型安全
- **导入检查**: 检查新增/修改的导入依赖
- **逻辑检查**: 分析变更代码的业务逻辑

### 3. 智能关联检查
- **依赖分析**: 检查变更文件的上下游影响
- **组件关联**: 如果修改了组件，检查使用该组件的页面
- **API关联**: 如果修改了API，检查调用该API的组件
- **样式关联**: 如果修改了样式，检查影响的页面布局

### 4. 动态功能验证 (仅变更部分)
- **组件测试**: 只测试修改的组件功能
- **页面验证**: 只访问受影响的页面
- **API测试**: 只测试变更的API调用
- **交互验证**: 只测试受影响的用户交互

### 5. 增量注释和文档
- **代码注释**: 为新增/修改的函数添加中文注释
- **组件文档**: 更新修改组件的Props和Events文档
- **API文档**: 更新变更API的接口说明
- **变更日志**: 记录本次变更的技术细节

### 6. 精准提交
- **暂存变更**: 只添加实际修改的文件
- **智能Commit**: 基于变更内容生成中文提交信息
- **影响范围**: 在commit信息中明确说明影响范围
- **关联提交**: 如需要，自动提交相关的配置文件

## Git Diff分析项目

### 变更文件识别
- **新增文件**: 新创建的组件、页面、API文件
- **修改文件**: 更新的组件、配置、样式文件
- **删除文件**: 移除的组件、页面、API文件
- **重命名文件**: 文件路径变更

### 变更类型判断
- **功能变更**: 新增功能、组件、页面
- **修复变更**: Bug修复、错误处理、兼容性
- **重构变更**: 代码优化、性能改进、结构调整
- **文档变更**: 注释、文档、说明更新

### 影响范围分析
- **组件影响**: 使用该组件的页面和组件
- **页面影响**: 该页面使用的组件和API
- **API影响**: 调用该API的组件和页面
- **配置影响**: 依赖该配置的模块和组件

## 增量检查项目详情

### 静态增量分析
- **语法检查**: 只检查变更文件的TypeScript语法
- **代码规范**: 验证变更代码的ESLint规则
- **依赖分析**: 检查变更文件的新增依赖
- **类型安全**: 分析变更代码的类型定义
- **逻辑审查**: 检查变更代码的业务逻辑

### 智能关联分析
- **上游依赖**: 变更文件依赖的其他文件
- **下游影响**: 依赖变更文件的其他文件
- **同级关联**: 同级文件的相互影响
- **间接影响**: 可能的间接影响范围

### 动态增量验证
- **组件测试**: 只测试修改的组件功能
- **页面测试**: 只访问包含变更组件的页面
- **API测试**: 只测试变更的API接口
- **交互测试**: 只测试受影响的用户交互

### 增量注释规则
- **新增函数**: 为新函数添加完整中文注释
- **修改函数**: 更新修改函数的注释
- **组件Props**: 为新增Props添加类型和说明
- **变更说明**: 在注释中说明变更原因

## Commit Message生成规则

### 类型识别 (基于变更内容)
- **feat**: 新增组件、页面、功能
- **fix**: 修复Bug、错误处理、兼容性
- **refactor**: 代码重构、性能优化
- **docs**: 注释、文档更新
- **style**: 样式调整、代码格式
- **test**: 测试相关文件
- **chore**: 配置、工具、构建文件

### 影响范围 (基于变更文件)
- **components**: 组件库相关变更
- **dashboard**: 监控仪表板相关
- **api**: API集成和服务相关
- **config**: 配置文件相关
- **pages**: 页面路由和布局相关

### 中文描述格式
```
类型(范围): 简洁描述变更内容

## 变更详情
- 主要变更点1
- 主要变更点2
- 影响范围说明

🤖 Generated with [Claude Code](https://claude.com/claude-code)

Co-Authored-By: Claude <noreply@anthropic.com>
```

## 质量标准

### 增量检查通过标准
- 变更文件无语法错误
- 变更逻辑正确无误
- 关联功能正常工作
- 新增注释规范完整
- 提交信息准确描述变更

### 检查失败处理
- 语法错误 → 提供修复建议
- 逻辑错误 → 指出问题位置
- 功能问题 → 提供调试方向
- 规范问题 → 给出修改建议

## 注意事项

- **无变更检查**: 如果没有Git变更，提示用户
- **大型变更**: 超过20个文件变更时提供分批建议
- **复杂变更**: 复杂逻辑变更提供详细分析
- **冲突处理**: 如有Git冲突，先解决冲突再执行
