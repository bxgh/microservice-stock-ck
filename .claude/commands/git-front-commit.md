# Smart Commit - 智能代码提交

## 命令描述

智能分析和提交前端代码，包含完整的质量检查和自动化流程。

## 执行流程

### 1. 静态代码分析 (语法/规范)
- 检查TypeScript语法错误
- 验证Vue组件规范
- 检查ESLint规则违规
- 分析代码复杂度和潜在问题
- 检查未使用的导入和变量

### 2. 动态功能检查 (页面/交互)
- 验证开发服务器运行状态
- 检查页面是否能正常渲染
- 测试组件交互功能
- 验证API调用是否成功
- 检查控制台错误和警告

### 3. 生成中文注释和文档
- 分析代码逻辑生成中文注释
- 为复杂函数添加说明
- 更新组件文档
- 生成API文档说明

### 4. 性能和安全性检查
- 检查潜在的内存泄漏
- 分析性能瓶颈
- 安全漏洞扫描
- 检查敏感信息泄露

### 5. 生成中文commit message
- 分析代码变更内容
- 自动识别变更类型（功能/修复/优化/文档等）
- 生成符合规范的中文提交信息
- 包含变更影响范围

### 6. 执行提交
- 添加相关文件到暂存区
- 使用生成的commit message提交
- 可选推送到远程仓库

## 使用方法

### 基本使用
```
智能分析和提交前端代码，包含质量检查和中文注释生成
```

### 指定检查范围
```
只检查和提交apps/frontend-web/src/components/目录的代码
```

### 详细模式
```
智能分析代码，生成详细报告，然后提交
```

## 检查项目详情

### 静态分析项目
- **语法检查**: TypeScript编译错误、语法问题
- **代码规范**: ESLint规则、代码风格
- **依赖分析**: 未使用的导入、循环依赖
- **复杂度检查**: 函数复杂度、嵌套层级
- **类型安全**: 类型推导、any类型使用

### 动态检查项目
- **页面渲染**: 组件是否正常显示
- **交互功能**: 按钮点击、表单提交
- **API集成**: 微服务调用是否正常
- **错误处理**: 异常情况处理
- **性能指标**: 加载时间、响应时间

### 注释生成规则
- **函数注释**: 功能描述、参数说明、返回值
- **组件注释**: 组件用途、Props说明、事件说明
- **复杂逻辑**: 算法说明、业务逻辑解释
- **API注释**: 接口用途、参数格式、返回格式

### Commit Message生成规则
- **类型识别**:
  - feat: 新功能、新组件、新页面
  - fix: Bug修复、错误处理
  - refactor: 代码重构、性能优化
  - docs: 文档更新、注释添加
  - style: 样式调整、代码格式
  - test: 测试相关
  - chore: 配置更新、工具优化
- **影响范围**: frontend/components/api/config等
- **变更描述**: 简洁明了的中文描述
- **技术细节**: 使用的技术、框架、工具

## 质量标准

### 通过标准
- 无语法错误和编译警告
- 页面正常渲染和交互
- 无明显的性能问题
- 无安全漏洞
- 代码符合项目规范

### 不通过处理
- 修复语法错误后重试
- 优化性能问题后重试
- 解决安全漏洞后重试
- 调整代码规范后重试

## 注意事项

- 确保开发服务器正在运行 (http://localhost:3001)
- 提交前建议手动快速验证功能
- 大型项目分析可能需要更多时间
- 如发现问题会自动中止并提供建议
