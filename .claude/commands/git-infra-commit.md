# Infrastructure Smart Commit - 基础设施智能提交

## 命令描述

基于Git diff快速扫描和提交基础设施配置变更，**仅处理CI/CD、Docker、脚本等基础设施文件**，避免业务代码干扰，实现精准的变更管理。

## 执行流程

### 1. 基础设施域扫描
- 执行 `git diff --name-only` 筛选基础设施变更文件
- **筛选规则**: 仅处理 `infrastructure/`、`scripts/`、`.github/` 等目录
- 跳过 `apps/`、`services/` 业务代码文件
- 识别变更类型：Docker/CI/CD/脚本/配置

### 2. 基础设施变更分析
- **Docker配置**: 检查Dockerfile和docker-compose文件
- **CI/CD流水线**: 验证GitHub Actions工作流配置
- **脚本检查**: 分析部署和维护脚本语法
- **环境配置**: 检查环境变量和配置文件

### 3. 影响范围评估
- **部署影响**: 评估变更对部署流程的影响
- **服务依赖**: 分析基础设施变更对运行服务的影响
- **环境一致性**: 检查多环境配置的一致性
- **安全审查**: 验证敏感信息暴露风险

### 4. 快速验证
- **Docker语法**: 验证Dockerfile和compose文件语法
- **YAML检查**: 验证CI/CD配置文件格式
- **脚本语法**: 检查Shell/Python脚本语法
- **权限检查**: 验证文件权限和可执行性

### 5. 精准提交
- **基础设施暂存**: 仅添加基础设施相关变更
- **影响标注**: 标注对服务和部署的影响
- **风险评估**: 提供变更风险等级和建议
- **回滚方案**: 提供变更失败时的回滚建议

## 使用方法

### 基本基础设施提交

提交基础设施配置变更
```

### 指定变更描述

```

### CI/CD流水线提交
```
修改了GitHub Actions工作流，优化了测试流程，请智能分析并提交
```

## 基础设施域扫描规则

### 变更文件筛选
```bash
# 筛选基础设施相关文件
git diff --name-only | grep -E "^(infrastructure/|scripts/|\.github/|docker-compose|Dockerfile|\.env\.example)"
```

### 基础设施分类
- **容器化**: Dockerfile, docker-compose*.yml, .dockerignore
- **CI/CD**: .github/workflows/, Jenkinsfile
- **部署脚本**: scripts/deploy/, scripts/setup/
- **配置文件**: infrastructure/, .env.example, Makefile
- **监控工具**: monitoring/, grafana/, prometheus/

### 变更风险等级
1. **高风险**: docker-compose.yml, 数据库配置, 安全证书
2. **中风险**: CI/CD流水线, 部署脚本, 环境变量
3. **低风险**: 监控配置, 文档更新, 工具脚本

## 特殊检查项目

### Docker配置检查
- **端口冲突**: 检查服务端口是否重复
- **卷挂载**: 验证数据卷路径有效性
- **网络配置**: 检查服务间网络连通性
- **镜像版本**: 验证Docker镜像版本一致性

### CI/CD流水线检查
- **语法验证**: YAML文件格式检查
- **密钥安全**: 检查敏感信息泄露风险
- **依赖关系**: 验证流水线步骤依赖
- **环境隔离**: 检查多环境配置差异

### 脚本安全检查
- **权限检查**: 验证脚本执行权限
- **路径安全**: 检查文件路径注入风险
- **命令注入**: 分析shell命令安全性
- **错误处理**: 验证异常处理逻辑

## 注意事项

- **业务代码忽略**: 自动忽略apps/和services/目录的业务代码
- **高风险变更**: Docker Compose和数据库配置需要特别谨慎
- **测试环境**: 基础设施变更建议先在测试环境验证
- **回滚准备**: 确保有完整的回滚方案

## 使用建议

### 推荐使用场景
- ✅ **Docker配置优化**: 容器化配置调整
- ✅ **CI/CD流水线更新**: 自动化部署流程改进
- ✅ **环境配置调整**: 开发/测试/生产环境配置
- ✅ **监控工具集成**: 日志、监控、告警配置

### 不推荐使用场景
- ❌ **业务逻辑变更**: 应用代码功能变更
- ❌ **数据库迁移**: 生产数据库结构变更
- ❌ **安全证书更新**: 生产环境证书更换

这个命令专注于基础设施配置的快速智能提交，确保DevOps变更的安全性和可追溯性。
