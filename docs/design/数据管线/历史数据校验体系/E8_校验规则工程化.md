# E8: 校验规则工程化(P2, 持续)

> 校验规则本身也是代码资产,长期看必须工程化。

## Story E8-S1: 规则声明式化

* 把分散的校验逻辑统一抽象成"规则"概念
* 每条规则有:ID、版本、描述、严重等级、适用范围
* 优先用 YAML / JSON 声明,Python 实现执行引擎
* 不必引入 Great Expectations 全家桶,自己写一个轻量框架更可控

## Story E8-S2: 规则单元测试

* 每条规则配套单测:正例(应通过)+ 反例(应命中)
* CI 跑通才能合入主干
* 规则修改时,通过测试结果对比新旧版本影响范围

## Story E8-S3: 规则版本管理

* 每次规则修改递增版本号
* `dq_findings` 和 `backfill_audit` 都记录"基于哪个版本的规则触发"
* 规则有 bug 时,可以批量定位"基于错误规则做的修复"并回滚
