# E5: 业务规则校验(P1)

> 这一块最容易被忽略,但报警里 80% 的"误报"都源于没处理好特殊品种。

## Story E5-S1: 建立特殊品种维度表

落地 4 张关键维度表:

* `dim_stock_status`:每只股票每个交易日的状态(正常 / ST / *ST / 退市整理 / 停牌 / 新股 N 日内)
* `dim_price_limit`:每只股票每个交易日的涨跌幅限制(±5% / ±10% / ±20% / ±30% / 无限制)
* `dim_corporate_action`:除权除息事件流水
* `dim_code_remap`:证券代码变更映射(借壳、重组场景)

## Story E5-S2: 涨跌幅校验

* 对每条日线记录,根据 `dim_price_limit` 校验涨跌幅
* 超限的标记为可疑,但**不要直接判错**——需结合是否新股、是否退市整理期等
* 复权日的涨跌幅校验必须用前一日复权后收盘价

## Story E5-S3: 复权因子对账

* 每周一次,根据 `dim_corporate_action` 中的分红派息事件,反算理论复权因子
* 与数据源提供的复权因子比对,差异超过 0.01% 触发人工 review
* 这是发现"复权数据被错误修订"的关键手段

## Story E5-S4: 特殊场景处理清单

按优先级落地特殊场景的校验规则:

1. 停牌一致性(已在 E3 完成)
2. ST 与涨跌幅限制
   - **重要提示**：ST 涨跌幅规则具有**强时效性**和**板块差异性**。
   - 创业板/科创板 ST 股票在注册制改革后为 ±20%。
   - 主板 ST/*ST 股票历史上为 ±5%，需特别注意交易所最新规（如主板 ST 涨跌幅限制未来调整为 ±10% 的生效节点）。
   - `dim_price_limit` 的生成逻辑必须是基于 `trade_date`、`板块` 和 `ST状态` 的三维动态计算。
3. 新股上市首日
4. 退市整理期
5. 除权除息日的连续性(避免误报为"价格断档")
6. 转债强赎/回售相关停牌
7. ETF 申赎清单一致性(如有需要)

## 验收标准

* 对历史上发生过的特殊事件(如某次借壳)进行回放,校验逻辑能正确识别不报错
* 涨跌幅超限的记录全部能解释(新股 / ST / 数据错误三选一)
* 复权因子对账每周生成报告
