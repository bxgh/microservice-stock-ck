# E7: 可观测性与 SLA(P2, 持续迭代)

> 没有度量就没有改进。这一块是长期事情,但要尽早开始。

## Story E7-S1: DQ 指标体系

每日固定时间产出 4 个核心指标:

* **完整性**:实际记录数 / 预期记录数,目标 ≥ 99.9%
* **一致性**:跨源比对一致率,目标 ≥ 99.5%
* **准确性**:行内校验通过率,目标 100%
* **及时性**:T+1 数据可用时间,目标 ≤ 早 9:00

每个指标按表、按数据源、按板块分别统计。

## Story E7-S2: 可视化看板

* 不必上 Grafana 全家桶,可以从简陋开始
* 第一版:每日邮件 / 微信推送 HTML 报告(沿用上面定义的研报风格)
* 第二版:本地 Web 服务 + 趋势图
* 第三版(如有必要):接入 Grafana / Superset

## Story E7-S3: 告警分级

定义清晰的告警级别和响应 SLA:

* **P0 紧急**(如完整性 < 95%):立即推送,1 小时内响应
* **P1 重要**(如跨源比对差异 > 5%):工作时间推送,当日响应
* **P2 一般**:汇总到日报,每周 review
* **P3 提示**:只入库,不推送

避免告警轰炸是这一步的关键——一旦团队对告警麻木,整个体系就废了。
