# E6: 内网重算节点对接规范

> 适用对象：内网计算节点 (Local Compute Node)
> 目标：监听云端修复信号，执行指标重算，并将结果同步回云端。

## 1. 业务流程 (Pull & Ack)

1.  **轮询 (Polling)**：内网节点每隔 60s 调用 `GET /recalc-signals/pending` 获取待处理任务。
2.  **锁定**：云端返回任务的同时，会自动将该任务标记为 `PROCESSING`，防止其他节点重复领取。
3.  **重算**：内网节点根据 `ts_code` 和 `start_date` ~ `end_date` 执行计算。
4.  **回传**：计算完成后，通过现有指标同步接口将结果写入云端。
5.  **确认 (Ack)**：调用 `POST /recalc-signals/ack`，告知云端处理完成。

## 2. API 接口详解

### 2.1 获取待重算信号
*   **接口**: `GET /api/v1/backfill/recalc-signals/pending`
*   **参数**: `limit` (默认 50)
*   **响应示例**:
    ```json
    {
      "count": 1,
      "data": [
        {
          "id": 1024,
          "ts_code": "600519.SH",
          "start_date": "2024-05-13",
          "end_date": "2025-01-20",
          "request_id": "BF-05051110-2049",
          "created_at": "2026-05-05T11:10:00"
        }
      ]
    }
    ```

### 2.2 确认重算完成
*   **接口**: `POST /api/v1/backfill/recalc-signals/ack`
*   **请求体**:
    ```json
    {
      "request_id": "BF-05051110-2049",
      "status": "COMPLETED",
      "notes": "指标重算成功，涉及 250 行记录"
    }
    ```
*   **响应**: `{"status": "success"}`

## 3. 注意事项 (Critical)

1.  **内存管理**：内网节点在处理长跨度（如 > 250 天）重算时，请务必采用分批读取模式，单次内存占用不得超过 128MB。
2.  **超时处理**：如果拉取任务后 30 分钟未 Ack，云端扫描脚本可能会将其重置为 `PENDING` 重新分发。
3.  **幂等性**：内网节点应确保计算逻辑是幂等的，即针对同一时间段的重算结果应保持一致。
4.  **失败处理**：若重算失败，Ack 时可将 `status` 设为 `FAILED` 并在 `notes` 中记录错误堆栈。
