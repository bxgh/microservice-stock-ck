# 历史数据修复与架构重构方案

## 1. 背景与目标
基于 `stock_kline_daily` 和 `stock_adjust_factor` 的全量审计结果，发现个股断档及因子错误。本方案旨在修复数据缺陷，并重构复权计算逻辑，确保系统高性能与数据一致性。

## 2. 核心任务分解

### 阶段一：数据疗愈 (Data Healing)
- **任务 A：K 线空洞回填**
    - 脚本：`scratch/backfill_kline_gaps.py`
    - 逻辑：读取 `audit_reports/report_B_missing_dates.csv`，定向补齐。
- **任务 B：因子盲区修复**
    - 脚本：`scratch/backfill_factor_errors.py`
    - 逻辑：处理 `critical_factor_errors.csv`，补齐缺失初始因子并校正异常值。

### 阶段二：写入机制加固 (Architecture Enforcement)
- **代码修改**：`app/services/market_data_service.py`
- **变更点**：`ON DUPLICATE KEY UPDATE` -> **`INSERT IGNORE`**。
- **目标**：确立历史数据不可变性。

### 阶段三：计算下沉与视图重构 (Calculation Offloading)
- **任务**：创建 `v_stock_kline_forward_adj` 视图。
- **公式**：$P_{adj} = P_{raw} \times F_{date} / F_{latest}$。
- **目标**：利用数据库引擎进行实时前复权计算。

### 阶段四：服务逻辑清理 (Refactoring)
- **清理项**：删除 Python 侧冗余的循环复权计算代码。
- **路由切换**：API 查询切换至视图。

## 3. 验收标准
- **性能**：10年历史数据复权查询耗时 < 200ms。
- **准确度**：与旧版逻辑计算结果误差 < 0.0001。
