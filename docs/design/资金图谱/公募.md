# 资金获取方案: 公募基金

## 底层数据
公募持仓数据通常滞后且仅在季报披露。日常估算需采用“间接指标”合成。

## 计算公式 (三因子合成)
```
公募活跃度 = 0.4 × 仓位变化Score + 0.3 × 新发申购Score + 0.3 × ETF场内份额Score
```

### 1. 仓位变化 Score
- **逻辑**: 基于卖方仓位测算, 用前5个交易日的仓位变化标准化。
- **数据源**: 中信证券、兴业证券每周一发布的卖方研报。

### 2. 新发申购 Score
- **逻辑**: 基于中基协月度新发数据, 日度用线性插值。
- **数据源**: 中国证券投资基金业协会 (每月15日左右发布上月数据)。
- **近似源**: Wind 的“基金市场规模”日频数据。

### 3. ETF 场内份额 Score
- **逻辑**: 所有股票型 ETF 的场内份额日变化。
- **数据源**: 沪深交易所每日 ETF 日报。
- **接口**: akshare 的 `fund_etf_category_sina()`。

# 1. 活跃度识别分 (用于主导资金识别, 20日窗口)
ActivityScore = (今日公募子因子加权后 - 20日均值) / 20日标准差 × Continuity(5d)

# 2. 状态屏温度计得分 (用于视觉显示, 20日/60日窗口)
Fund_Score = Z(公募活跃度, 20d)

## 仓位警戒线 (状态屏"公募仓位"指标)
- **绿灯 (正常)**: 仓位 < 80% (机构有充足子弹, 向上空间大)
- **黄灯 (预警)**: 80% ≤ 仓位 ≤ 88% (进入减速带, 仍可维持趋势)
- **红灯 (危险)**: 仓位 > 88% (历史极致高位, 警惕机构被动减仓风险)

## 校准机制
每季度公募季报披露后, 用真实数据校准日度估算的偏差。如果发现某个因子的预测力衰减, 应调整权重。
