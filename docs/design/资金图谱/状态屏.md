# 状态屏指标的完整计算手册

状态屏是整个观测体系的入口,它的指标必须**精确可计算、数据可获取、阈值可验证**。下面把状态屏的所有指标拆解到最底层,提供完整的计算方法、数据来源和阈值设置。

## 状态屏的指标地图

先回顾一下状态屏包含的所有指标,然后逐一深入:

**第一区:市场状态总览**
- 当前主导资金识别
- 市场状态(A/B/C/D)判定
- 状态持续天数
- 状态切换概率

**第二区:九类资金温度计**
- 国家队、北向配置盘、北向交易盘、公募、量化、险资、游资、散户、产业资本

**第三区:六大警戒灯**
- 流动性环境、公募仓位、量化拥挤度、散户情绪、北向背离、政策密度

下面按这个顺序逐一展开。

---

## 第一区:市场状态总览的计算方法

### 指标一:当前主导资金识别

**核心思路:** 计算每一类资金的"活跃度评分",评分最高的就是当前主导资金。

**计算公式:**

每一类资金的活跃度评分 = 标准化后的资金流向强度 × 持续性权重

```
ActivityScore_i = Z(NetFlow_i, 20d) × Continuity_i

其中:
Z(NetFlow_i, 20d) = (今日净流入 - 过去20日均值) / 过去20日标准差
Continuity_i = (过去5日中同向流入的天数) / 5
```

**为什么这样设计:**
- Z-score标准化:不同资金的规模差异极大(国家队单日可能100亿,游资可能5亿),用标准化消除规模差异,使各类资金可比
- 20日窗口:平衡灵敏度与稳定性,既不会被单日异常干扰,也不会过于滞后。**注:此窗口用于各资金横向对比识别主导权,各资金子温控分可采用更长窗口(如60日)以追求显示稳定性。**
- 持续性权重:防止偶然的单日异动被误判为"主导资金"

**九类资金的具体净流入计算:**

| 资金类别 | 净流入计算方法 | 数据源 |
|---------|--------------|--------|
| 国家队 | 八大核心汇金系ETF净申购金额合计 (含300、50、500、1000) | 上交所/深交所ETF日报 |
| 北向配置盘 | 配置类托管行(汇丰、摩根大通、花旗、渣打、德意志、法巴)的持股市值变化 | HKEX CCASS |
| 北向交易盘 | 交易类托管行(高盛、摩根士丹利、瑞银、巴克莱)的持股市值变化 | HKEX CCASS |
| 公募 | 偏股混合型基金净值×平均仓位变化 + 当日新发规模估算 | 中信证券/兴业证券测算 |
| 量化 | (中证500+1000)主力合约成交额 + 量化策略平均超额收益反推 | Wind/Choice |
| 险资 | 月度更新,日度用上月数据线性插值 + 重大举牌公告即时更新 | 国家金融监管总局 |
| 游资 | 知名游资席位龙虎榜净买入合计 | 交易所龙虎榜 |
| 散户 | 两融余额日变化 + 全A小单占比变化 | 沪深交易所 |
| 产业资本 | 大股东+董监高增减持金额合计 | 交易所重大股东公告 |

**输出:** 取活跃度评分最高的资金类别作为"当前主导资金"。当评分最高的两类资金分数差距小于0.3σ时,显示为"双主导"或"过渡期"。

### 指标二:市场状态(A/B/C/D)判定

这是状态屏最核心的判定。**采用"加权得分制"而不是"硬规则触发制"**——因为硬规则容易在边界值附近频繁切换,产生噪音信号。

**计算逻辑:**

每个状态都有一组特征指标,每个指标对该状态的贡献度不同。算法计算每个状态的"匹配得分",得分最高的状态就是当前状态。

**状态A(机构趋势市)的匹配得分:**

```
Score_A = w1 × I(北向配置盘5日累计>50亿) 
        + w2 × I(80% < 公募仓位 < 88%)
        + w3 × I(大盘相对小盘强度上行)
        + w4 × I(行业轮动速度<0.5,即主线明确)
        + w5 × I(DR007稳定且<2.5%)
        + w6 × I(量化拥挤度<历史60%分位)

权重: w1=0.25, w2=0.20, w3=0.15, w4=0.15, w5=0.10, w6=0.15

I() 是指示函数,满足条件取1,不满足取0
```

**状态B(游资题材市)的匹配得分:**

```
Score_B = w1 × I(两融占比>9%)
        + w2 × I(中证1000相对沪深300强度上行)
        + w3 × I(行业轮动速度>0.7,即题材分散)
        + w4 × I(连板高度≥5板)
        + w5 × I(炸板率<25%)
        + w6 × I(游资席位活跃度>30家/日)

权重: w1=0.15, w2=0.15, w3=0.20, w4=0.20, w5=0.15, w6=0.15
```

**状态C(量化漂移)的匹配得分:**

```
Score_C = w1 × I(微盘股/中证1000 20日涨幅差>5%)
        + w2 × I(指增超额收益滚动20日>5%)
        + w3 × I(IC/IM贴水年化>10%)
        + w4 × I(行业轮动速度>0.8,极快)
        + w5 × I(量化拥挤度>历史85%分位)

权重: w1=0.25, w2=0.20, w3=0.20, w4=0.10, w5=0.25
```

**状态D(避险防御市)的匹配得分:**

```
Score_D = w1 × I(北向连续3日净流出)
        + w2 × I(公募仓位下降>2%)
        + w3 × I(红利低波相对主流指数强度上行)
        + w4 × I(DR007上行或美债收益率快速上行)
        + w5 × I(成交额萎缩至8000亿以下)
        + w6 × I(50ETF期权PCR>1.2)

权重: w1=0.20, w2=0.15, w3=0.15, w4=0.20, w5=0.15, w6=0.15
```

**输出规则:**
- 取最高分的状态为当前状态
- 最高分必须>0.55才确认状态(防止低置信度判定)
- 最高分与第二高分之差>0.15才确认稳定状态,否则标记为"过渡期"

**验证方法:** 用过去3年的数据回测,看四种状态的判定结果是否符合事后市场表现。**所有权重和阈值都必须用历史数据校准,不能拍脑袋**。

### 指标三:状态持续天数

**计算方法:** 从最近一次状态切换的日期开始,累计交易日天数。

**状态切换的定义:** 当前状态的得分连续3个交易日低于新状态的得分,且新状态得分>0.55,确认状态切换。

**为什么需要"连续3日"确认:** 防止单日波动导致的状态误切换。状态判定要追求"稳定性"而非"灵敏度"。

**显示逻辑:**
- 0-5天:橙色,表示新状态未稳固
- 6-20天:绿色,稳定状态
- 21-45天:黄色,需要警觉(状态可能进入末期)
- 46天以上:红色,状态过度延长,切换风险高

### 指标四:状态切换概率

这是状态屏最有前瞻性也最难计算的指标。给出两种实现路径,从简到繁:

**简单版本:基于历史频率的统计估计**

收集过去5年所有状态切换的样本(假设有100次状态切换),然后统计:
- 当前状态的"对手状态"得分上升到什么程度时,5天内切换的频率有多高?

具体计算:

```
SwitchProb_5d = 历史样本中,当前指标组合5天内发生切换的比例

具体步骤:
1. 计算当前所有四个状态得分
2. 找出过去5年中,得分组合最相似的N=20个样本(用欧氏距离)
3. 统计这20个样本中,5天内确实发生状态切换的比例
4. 这个比例就是切换概率
```

**进阶版本:Logistic回归模型**

用过去3年的日度数据训练一个二分类模型:
- 输入:当前各状态得分、关键指标的边际变化率、波动率指标
- 输出:5个交易日内发生状态切换的概率

模型特征建议包括:
- 当前主导状态的得分
- 第二高状态的得分
- 关键警戒指标的Z-score
- 成交额20日变化率
- 北向资金近5日方向一致性
- VIX(中国版波动率指数)的变化

**显示逻辑:**
- <30%:绿色,状态稳定
- 30-50%:黄色,需要关注
- 50-70%:橙色,显著切换风险
- >70%:红色,准备切换

## 第二区:九类资金温度计的计算方案

每一类资金的温度计均基于 Z-score 框架进行标准化度量。为了保持手册简洁，详细的数据源定义、计算公式及校准机制已切分为独立方案文件，请点击链接查看。

- **[国家队](acquisition_schemes/national_team.md)**：基于汇金系 ETF 净申购估算。
- **[北向资金](acquisition_schemes/northbound_funds.md)**：区分配置盘与交易盘，基于 CCASS 托管行明细聚合。
- **[公募基金](acquisition_schemes/mutual_funds.md)**：三因子合成（仓位测算 + 新发规模 + ETF 份额）。
- **[量化基金](acquisition_schemes/quant_funds.md)**：四因子合成（指增超额 + 小微盘强度 + 期货贴水 + 成交集中度）。
- **[险资](acquisition_schemes/insurance_funds.md)**：基于月度披露数据线性化及事件驱动更新。
- **[游资](acquisition_schemes/hot_money.md)**：基于龙虎榜活跃席位库的活跃度打分。
- **[散户](acquisition_schemes/retail_investors.md)**：四因子合成（两融 + 小单占比 + 开户数 + 搜索指数）。
- **[产业资本](acquisition_schemes/industrial_capital.md)**：基于上市公司股东增减持公告统计。

## 第三区:六大警戒灯的计算方法

警戒灯是状态屏中**最具触发性**的指标。每个灯都是一个二元判断:绿(正常)、黄(警觉)、红(危险)。

### 警戒灯一:流动性环境

**核心指标:** DR007(银行间7天质押式回购利率)+ 10年期国债收益率

**判定规则:**

```
绿灯条件(全部满足):
- DR007 < 2.5%
- 10年期国债收益率 < 2.8%
- M1同比 > 0%
- 央行公开市场操作近5日净投放

黄灯条件(任一满足):
- 2.5% < DR007 < 2.8%
- 10年期国债收益率快速上行(20日上行>30bp)
- M1同比转负

红灯条件(任一满足):
- DR007 > 2.8%
- 10年期国债收益率20日上行>50bp
- 央行连续大额回笼
```

**数据源:**
- DR007:中国货币网`http://www.chinamoney.com.cn/`
- 10年期国债:中国债券信息网,或Wind/Choice
- M1:国家统计局、人民银行

### 警戒灯二:公募仓位

**核心指标:** 偏股混合型基金平均仓位

**判定规则:**

```
绿灯:仓位 < 80%(还有加仓空间)
黄灯:80% ≤ 仓位 ≤ 88%(中性偏高)
红灯:仓位 > 88%(高位预警,接近上限)
```

**数据源:**
- 中信证券、兴业证券、招商证券每周一发布的卖方研报
- Wind的"基金仓位测算"
- 简化:可以追踪几只代表性偏股混合基金的净值变化反推

**特殊提示:** 当公募仓位连续3周下降时,即使绝对值还在黄灯区,也应该弹出"机构调仓中"的提示。

### 警戒灯三:量化拥挤度

**核心指标(三因子合成):**

```
拥挤度Score = 0.4 × 指增超额z + 0.3 × 小微盘强度z + 0.3 × 期货贴水z

每个因子用过去3年数据计算分位数

判定规则:
绿灯:综合分位 < 60%(拥挤度低)
黄灯:60% ≤ 综合分位 ≤ 85%(中度拥挤)
红灯:综合分位 > 85%(高度拥挤,流动性危机风险)
```

**数据源:** 同温度计五

**警示机制:** 当红灯持续超过5个交易日,系统应触发"小微盘流动性预警",建议减少小盘暴露。

### 警戒灯四:散户情绪

**核心指标:** 散户情绪综合指数

```
散户情绪 = 0.4 × 两融余额z + 0.3 × 新增开户z + 0.3 × 搜索指数z

判定规则(注意散户情绪通常作为反向指标):
绿灯:综合Z在 [-1, +1] 之间(中性区域,正常)
黄灯:综合Z 在 [-2, -1] 或 [+1, +2] (情绪偏离)
红灯:综合Z > +2 或 < -2 (极端情绪,反向风险)
```

**重要说明:** 散户情绪极致高时(>+2σ),要警惕顶部;散户情绪极致低时(<-2σ),要关注底部机会。系统在红灯时应该弹出反向操作提示。

### 警戒灯五:北向背离

**核心指标:** 配置盘和交易盘的方向背离

**判定逻辑:**

```
计算配置盘和交易盘过去5日累计净流入

绿灯条件:配置盘和交易盘同方向(都正或都负)
黄灯条件:一方接近0,另一方明显
红灯条件:配置盘和交易盘明显背离(方向相反且各自|金额|>20亿)
```

**红灯的二级提示:**

```
红灯类型A:配置盘+,交易盘-
含义:长线资金在底部布局,短线资金在避险
通常意味:潜在底部,可适度布局核心资产

红灯类型B:配置盘-,交易盘+  
含义:长线资金在撤退,短线资金在博弈反弹
通常意味:潜在顶部,警惕,准备防御
```

### 警戒灯六:政策密度

**核心指标:** 政策催化指数

**计算逻辑:**

```
1. 维护一个政策关键词库
   - 产业政策:新质生产力、低空经济、AI、机器人、半导体、新能源等
   - 监管政策:并购重组、市值管理、退市、IPO、减持等
   - 货币政策:降息、降准、MLF、公开市场操作等
   - 财政政策:特别国债、专项债、减税等

2. 每日爬取关键媒体
   - 新华社、人民日报
   - 发改委、证监会、金融监管总局
   - 央行、财政部
   - 工信部、科技部

3. 统计每日各关键词的提及次数

4. 政策密度Score = 关键词总提及次数与30日均值的偏离Z值

判定规则:
绿灯:Score 在 [-1, +1] (政策正常)
黄灯:+1 < Score < +2 (政策密集)
红灯:Score > +2 (政策爆发,可能催化大行情)
```

**特殊处理:** 红灯时应该按行业聚合显示,标注"今日政策催化最多的3个行业",方便快速定位题材。

**数据源:**
- 各部委官网(需爬虫)
- 新华社新闻列表
- 简化方案:订阅Wind的"政策事件"数据,或使用ChatGPT/Claude API做关键词提取
