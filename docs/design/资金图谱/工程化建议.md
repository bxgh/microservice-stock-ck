# 数据获取工程化建议与更新频率

## 数据源分层
根据“成本与稳定性”进行的推荐组合。

### 第一层: 免费开源
- **akshare**: 支持 90% 基础数据。
- **Tushare Pro**: 基础行情与财务数据。
- **交易所官网**: 权威权重、ETF 份额、龙虎榜。
- **中国货币网/债券网**: DR007、国债利率。
- **百度指数**: 搜索数据。

### 第二层: 商业付费 (中等成本)
- **同花顺 iFinD / 东财 Choice**: 适合专业投资者的全面数据。
- **专属平台**: 好买基金网、私募排排网 (针对量化/私募)。

### 第三层: 专业终端
- **Wind 金融终端**: 机构标配。
- **CCASS 专业数据**: 针对北向资金的深度明细。

## 数据更新频率调度

| 频率 | 指标内容 |
|------|----------|
| **实时 (秒级)** | 全 A 成交额、指数、涨停股数、北向实时流入 |
| **日度 (收盘后)** | ETF 申购、龙虎榜、两融余额、期货基差 |
| **T+1 (次日)** | CCASS 详细数据、北向托管行明细 |
| **周度** | 卖方公募仓位、新增开户、政策事件聚合 |
| **月度** | 中基协数据、险资运用、M1/M2 |

## 数据质量保障 (QA)
- **双源校验**: 关键指标 (如北向) 应同时使用多个源, 差异 >5% 时告警。
- **异常处理**: 失效时自动切换备用源, 标记数据为“待验证”, 并通过 IM 推送。

## 实施路径建议
1. **第一阶段 (1周)**: 搭建数据底座 (MySQL), 拉取 akshare 历史数据。
2. **第二阶段 (1周)**: 实现温度计 Z-score 计算与实时可视化。
3. **第三阶段 (1周)**: 实现警戒灯判定与告警推送。
4. **第四阶段 (1周)**: 实现 A/B/C/D 状态加权得分并回测校准权重。
5. **后续迭代**: 持续调整阈值, 每季度校准一次权重。
