## 1. 核心架构设计

为了保证盘后数据的稳定性与准确性，系统采用了 **多源冗余同步架构**：

| 层级 | 数据项 | 来源 (Primary) | 来源 (Fallback) | 计算/触发点 |
| :--- | :--- | :--- | :--- | :--- |
| **ODS** | 个股 K 线 | **Tushare** | BaoStock | stock-manager (20:15) |
| **ODS** | 指数 K 线 | **Tushare** | AkShare | stock-manager (20:15) |
| **ODS** | 申万/概念指数 | **AkShare** | - | akshare-api (19:15) |
| **ADS L1**| 市场全景 | - | - | IndicatorService (计算) |
| **ADS L2**| 结构分析 | - | - | StructuralAnalyzer (计算) |

## 2. 关键技术突破

### 2.1 Tushare 优先与 Failover 机制
由于 BaoStock 在盘后高峰期经常出现连接超时，我们将全市场 K 线同步切换为 **Tushare 优先** 模式。
- **优势**: Tushare 支持一次性拉取 5500+ 股票，效率提升 10 倍以上。
- **机制**: 若 Tushare 调用失败，自动回退到异步的 `BaoStock` 同步流程，确保数据不丢失。

### 2.2 128MB 限制下的内存优化 (OOM 修复)
全市场 5500+ 只股票的复权计算（250 日窗口）在 128MB 容器中会触发 OOM。
- **方案**: 引入 **分批处理迭代器** (Batching)。
- **实现**: 每 500 只股票作为一个计算批次，释放内存后再处理下一批，成功将内存占用稳定在 80MB 以内。

### 2.3 中证全指 (000985.SH) 口径对齐
统一全系统核心指数口径，废弃不稳定的 `000985.CSI` 定义，改为 Tushare 与主流通用的 `000985.SH`。

## 3. 指标体系字典

### 3.1 ADS L1 (市场全景)
- `idx_winda_close`: 对应中证全指，反映 A 股整体水位。
- `market_breadth`: 站上 20 日线股票占比，反映赚钱效应。

### 3.2 ADS L2 (结构分化)
- **行业旋转**: 申万一级行业近 5 日/20 日排名变动，识别主线轮动。
- **风格分化**: 计算“大盘价值/小盘成长”等 17 类因子的表现。
- **概念脉冲**: 监控同花顺热门概念板块的爆发力。

---

## 4. 自动化运维计划 (Cron)

- **19:15**: 行业/概念/风格数据同步 (AkShare) -> 触发 L2 计算。
- **20:15**: 全市场个股/指数 K 线同步 (Tushare) -> 计算市场广度 -> 生成 L1 看板。
