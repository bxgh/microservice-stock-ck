# 第 8 章 · 认知训练系统 (总结、运营与评估)

## 1. 训练节奏的产品化

把这套体系变成**可坚持**的关键,是节奏设计:

| 时间 | 动作 | 耗时 |
|---|---|---|
| 工作日 09:00-09:25 | 晨间预判 | 5 分钟 |
| 工作日 15:00-18:00 | 每日日记 | 2 分钟 |
| 工作日 任何时间 | 决策日志 | 按需, 3 分钟/次 |
| T+5 晚 18:00 | 中期反思 push | 2 分钟 |
| T+20 晚 18:00 | 终期反思 push | 2 分钟 |
| 周日 上午 09:00 | 系统推送候选失误 | 被动 |
| 周日 任何时间 | 周末复盘 | 15-20 分钟 |
| 每月 1 号 | 月度报告生成 | 查看 5 分钟 |

**总成本: 工作日每天 7 分钟 + 周末 20 分钟**。

---

## 2. 6 个月使用路径

| 阶段 | 时长 | 用户感受 | 系统输出 |
|---|---|---|---|
| **冷启动** | 1-30 天 | 写起来累,样本不足 | 只统计基础准确率,不出诊断 |
| **觉醒期** | 30-60 天 | 看到校准曲线时开始震惊 | 出现首个个人盲区识别 |
| **应用期** | 60-120 天 | 避坑清单首次实时拦截决策 | 准确率出现初步上升 |
| **稳定期** | 120-180 天 | 形成"看到 X 必查 Y"的反射 | 出现明显进步斜率 |

---

## 3. 风险与避坑 (注意点)

1. **强制提交弹性**: 改为"过期标灰"+"昨日补录扣分"机制, 避免用户反感。
2. **样本量噪声**: < 30 个样本不出准确率, 显示"数据积累中"。
3. **延迟反思窗口**: T+5 push 后给 3 天窗口, 过期影响月报评分。
4. **4 象限分类纠偏**: 允许用户在月度报告中手动 override 自动评定的结果。
5. **心理平衡**: 报告中强制每条负面发现配 1 条进步信号。

---

## 4. 里程碑计划

| 节点 | 交付 |
|---|---|
| **D+1** | 5 张训练表 DDL 上线 |
| **D+3** | 晨间预判页 + 自动校验 SQL |
| **D+5** | 每日日记 + 决策日志页 |
| **D+7** | T+5 / T+20 push 机制 |
| **D+10** | 周末复盘页 + 候选失误筛选 |
| **D+14** | 认知画像基础版(准确率 + 4 象限) |
| **D+21** | 校准曲线 + 月度报告模板 |
| **D+30** | 个人避坑清单实时触发拦截 |

---

## 5. 度量指标

- **完成率**: 晨间预判(≥80%)、每日日记(≥70%)、周末复盘(≥70%)。
- **改善度**: 准确率从 50% → 60%、校准曲线 RMSE 下降 30%、决策证据数从 1.5 → 3.0。
- **稳定性**: 校验 SQL 跑批成功率 ≥ 99%、仪表盘加载 < 800ms。

---

## 总结 (产品哲学)

> **数据让你看到市场,训练系统让你看到自己。**

第 8 章解决"我的认知长什么样"。90% 的散户失败是因为没有反馈闭环。这套系统的全部价值在于:**逼用户看清自己的判断模式**,然后用客观数据慢慢替换掉自欺欺人的叙事。
