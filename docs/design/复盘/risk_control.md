# 风险控制与工程实践

## 数据采集类

1. **接口字段漂移**: 采集层必须有字段映射层 + 单元测试。
2. **限频与封 IP**: 必须有 sleep + 重试 + 指数退避策略。
3. **数据时效错位**: 主任务建议 T 日 19:30 启动, 部分数据(两融/ETF份额) T+1 上午回填。
4. **北向数据退化**: 2024-08-19 后停止披露盘中实时数据, 仅可使用日终数据。

## 数据质量类

1. **行业分类兼容**: 申万一级 2021 年调整(28->31), 处理历史数据需注意映射。
2. **复权口径**: 估值用不复权、走势用前复权、回测用后复权。严禁混用。
3. **ST 与新股处理**: 连板、赚钱效应等指标应剔除上市 < 60 日新股及 ST 股。
4. **复权因子滞后**: 历史数据需定期根据最新复权因子进行回溯调整。

## 指标计算类

1. **历史基线长度**: 分位数指标至少需要 3-5 年数据, 10 年最佳。
2. **极端值污染**: 行业 PE 需考虑用中位数或剔除亏损股后再计算。
3. **样本变化**: 指数成分股调整需动态跟随, 避免幸存者偏差。

## 工程要求

1. **幂等性**: 任务必须支持重跑, 使用 `INSERT ... ON DUPLICATE KEY UPDATE`。
2. **数据血缘**: 派生指标需关联原始数据采集时间戳。
3. **自动化告警**: 采集失败或指标异常突变(如成交额翻倍)需触发告警。
