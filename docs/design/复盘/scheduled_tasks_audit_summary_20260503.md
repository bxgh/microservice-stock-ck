# 定时任务执行情况深度审计总结 (2026-05-03)

## 1. 总体评估
当前系统定时任务体系已建立，能够覆盖数据采集、指标计算及服务间协同。但由于调度配置缺失“交易日过滤”逻辑，且核心代码存在 SQL 参数匹配 Bug，导致**最近一个交易日 (2026-04-30) 及昨日 (2026-05-02) 的 L2 级深度指标计算全部失效**。

## 2. 关键发现与问题诊断

### 2.1 核心 Bug 记录 (P0 级)
- **现象**: `stock-manager` 日志显示 `【定时任务】同步流水线崩溃: not all arguments converted during string formatting`。
- **定位**: `IndicatorService.calculate_l2_indicators_full` 方法在执行 `insert_industry_sql` 时，SQL 仅需 1 个参数，但代码传入了 2 个。
- **后果**: 导致 L2 行业、概念、风格因子指标无法生成，数据链路在 ODS 层完成后即中断。
- **解决**: 已于 2026-05-03 11:37 完成代码修正。

  - K 线及 L1 全景数据：**正常**。
  - L2 深度指标（行业/概念/因子）：<span style="color:red">**完全缺失 (0 记录)**</span>。
- **2026-05-03 (今日)**:

### 2.3 调度设计缺陷
- **缺乏节假日感知**: 任务调度器目前仅基于固定时间触发，未集成 `CalendarService` 进行交易日判定，导致非交易日产生无效的系统负载和错误日志。
