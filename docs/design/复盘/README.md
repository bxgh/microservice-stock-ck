# A 股盘后数据观察体系 · 总纲

## 背景

为支持 A 股盘后行情复盘,需建立一套结构化的**原始数据采集需求**与**派生指标输出规范**,作为后续数据接入与计算实现的输入。

## 目标

- 明确盘后分析所需的原始数据类型、覆盖范围、更新频率与数据时效
- 定义最终对外输出的指标清单(分层、字段、含义)
- 沉淀实施过程中的关键风险与避坑要点

## 范围

- 单日盘后(15:00 之后)可获取的市场数据,以日频为主
- A 股全市场(沪深北),以及与 A 股决策强相关的港股、海外、商品、汇率、利率数据

## 非目标

- 不覆盖日内 Tick / 分钟级实时数据采集
- 不涉及量化策略回测、信号生成与下单执行
- 不涉及具体数据源选型与接口实现(由实施阶段决定)

---

## 技术依赖

- 关系型数据库或列式存储(承载结构化指标表)
- 对象存储或本地 Parquet(承载原始行情快照)
- 任务调度系统(支持依赖编排与重试)
- 监控告警系统(数据质量与采集状态)

## 里程碑

- **M1**: E1-S1 ~ E1-S4 数据接入完成,可输出 L1 + L3 + L4 指标 (**部分已实现**)
- **M2**: E1-S5 ~ E1-S7 接入,补齐 ETF / 估值 / 公告,完整 9 层指标
- **M3**: E1-S8 ~ E1-S9 接入,跨市场与衍生品全覆盖
- **M4**: 历史数据回补至 10 年,所有分位数指标达到稳态

## 度量指标

- 数据采集成功率 > 99.5%
- 关键指标计算延迟 < T 日 20:00(行情类) / T+1 日 10:00(财报类)
- 历史数据完整性: 近 5 年缺失日 < 5 天
- 字段空值率: 核心指标 < 0.5%, 扩展指标 < 5%
