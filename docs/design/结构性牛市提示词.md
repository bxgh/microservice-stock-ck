# Claude Code 开发需求：A股结构性牛市监控系统

## 项目背景与目标

**核心问题**：当前A股处于结构性牛市（新旧经济极度分化、马太效应显著），但这一结构何时会持续、何时会发生扭转，需要一套量化监控系统持续观测。

**目标**：开发一个独立运行的Python数据仪表盘，实时/定期计算一组关键指标，直观显示结构性牛市的"健康度"与"扭转风险"，并在关键信号出现时触发预警。

---

## 核心监控指标体系

系统需计算并展示以下五个维度的指标，每个维度包含1~3个具体量化指标。

---

### 维度一：市场分化程度（核心指标）

**含义**：衡量市场内部分化是否仍在持续。分化扩大=结构性牛市持续；分化收窄=可能扭转。

#### 指标 1.1：行业收益率离散度（Sector Return Dispersion）
- **计算方法**：过去20个交易日内，申万31个一级行业指数日收益率的横截面标准差，再取20日均值
- **数据来源**：申万行业指数（可用 `akshare` 获取：`sw_index_daily`）
- **判断逻辑**：
  - 离散度 > 历史75%分位 → 分化加剧，结构性特征强烈（绿色）
  - 离散度 25%~75%分位 → 分化正常，结构维持（黄色）
  - 离散度 < 历史25%分位 → 分化收窄，潜在扭转信号（红色）

#### 指标 1.2：新旧经济相对强度比（Growth/Value Ratio）
- **计算方法**：
  - 新经济篮子 = 科技（电子+计算机+通信+国防军工） + 新能源（电力设备） + 生物医药 的等权指数
  - 旧经济篮子 = 房地产 + 建筑材料 + 白酒（食品饮料子行业） + 银行 的等权指数
  - 比值 = 新经济篮子近60日累计收益 / 旧经济篮子近60日累计收益
- **判断逻辑**：
  - 比值 > 1.2 → 新经济显著占优，马太效应强（绿色）
  - 比值 0.9~1.2 → 相对均衡（黄色）
  - 比值 < 0.9 → 旧经济相对跑赢，结构可能扭转（红色）

#### 指标 1.3：涨跌家数比趋势（Market Breadth）
- **计算方法**：沪深两市每日上涨家数 / (上涨家数 + 下跌家数)，取20日移动平均
- **数据来源**：`akshare` 的 `stock_advance_decline` 或大盘情绪数据
- **判断逻辑**：
  - 趋势线斜率 > 0（持续改善）且绝对值 > 0.55 → 上涨扩散，结构可能均衡化（黄色/红色预警）
  - 趋势线斜率 < 0（持续恶化）且绝对值 < 0.45 → 下跌收窄到少数股，分化加剧（绿色）
  - 注意：此指标与结构性牛市的关系是**反向的**——宽度越差，结构性越强；宽度改善，结构可能弱化

---

### 维度二：马太效应强度（龙头集中度）

**含义**：衡量强者是否在持续吸筹，市值和资金是否持续向头部集中。

#### 指标 2.1：沪深300 vs 中证1000相对收益
- **计算方法**：沪深300近60日累计涨幅 - 中证1000近60日累计涨幅
- **数据来源**：`akshare` 的 `index_zh_a_hist`，代码分别为 `000300` 和 `000852`
- **判断逻辑**：
  - 差值 > +10% → 大盘龙头显著跑赢小盘，马太效应强（绿色）
  - 差值 -5%~+10% → 正常区间（黄色）
  - 差值 < -5% → 小盘跑赢大盘，资金扩散至非龙头，结构弱化信号（红色）

#### 指标 2.2：市值集中度（Top10%股票市值占比变化）
- **计算方法**：沪深全市场按市值排序，前10%公司的市值之和 / 全市场总市值，观察其近90日变化趋势
- **数据来源**：`akshare` 的 `stock_zh_a_spot_em`（实时行情，含市值）
- **判断逻辑**：
  - 占比环比上升 → 集中度提高，马太效应持续（绿色）
  - 占比环比下降且连续3周 → 集中度降低，资金扩散（红色预警）

---

### 维度三：增量资金流向（聪明钱方向）

**含义**：机构和外资的流向是结构性牛市最重要的维持力量，一旦发生方向性转变，是最可靠的扭转信号。

#### 指标 3.1：北向资金净流入趋势
- **计算方法**：
  - 过去20日北向资金净流入总额（正=流入，负=流出）
  - 过去20日北向资金流入的行业集中度（是否集中在新经济板块）
- **数据来源**：`akshare` 的 `stock_hsgt_fund_flow_summary_em`
- **判断逻辑**：
  - 20日净流入 > +200亿 且集中在科技/新能源 → 聪明钱持续涌入（绿色）
  - 20日净流入 -200亿~+200亿 → 中性（黄色）
  - 20日净流出 > 200亿 且连续3周 → 外资撤退，结构性扭转高风险（红色）

#### 指标 3.2：主力资金行业分布
- **计算方法**：过去10日，各主要行业ETF（科技/新能源/医药/地产/消费）的资金净流入排名变化
- **数据来源**：`akshare` 的 `fund_etf_fund_flow_ten_em`
- **判断逻辑**：观察科技类ETF是否持续净流入，地产/消费类ETF是否出现大规模净流入（若是，结构扭转预警）

---

### 维度四：跨市场联动强度

**含义**：A股科技板块与美股科技的相关性反映全球叙事一致性；相关性减弱可能意味着A股独立叙事的形成或断裂。

#### 指标 4.1：A股科技 vs 纳斯达克100相关系数
- **计算方法**：科创50指数日收益率 与 纳斯达克100（QQQ）日收益率的过去60日滚动相关系数
- **数据来源**：科创50用 `akshare` 的 `index_zh_a_hist`（代码 `000688`）；纳斯达克100用 `akshare` 的 `index_us_stock_sina`（或yfinance获取QQQ）
- **判断逻辑**：
  - 相关系数 > 0.6 → 全球共振，AI/科技叙事一致性强（绿色）
  - 相关系数 0.3~0.6 → 部分联动（黄色）
  - 相关系数 < 0.3 或负相关 → 联动断裂，需分析原因（黄色/红色）

#### 指标 4.2：美股传导有效性（滞后相关）
- **计算方法**：纳斯达克100今日涨跌 vs 科创50明日涨跌的相关系数（过去60日），衡量"美股领先A股"的传导效率
- **判断逻辑**：传导相关系数显著下降可能意味着A股逐渐脱离全球科技叙事

---

### 维度五：政策信号强度（文本分析）

**含义**：政策是A股结构性牛市最根本的驱动力，政策措辞的变化是最前瞻的扭转信号。

#### 指标 5.1：政策措辞监控（半自动）
- **实现方式**：
  1. 爬取新华社、人民日报、国家发改委官网近7天新闻标题
  2. 对标题和摘要进行关键词频率统计
  3. 展示"支持类关键词"（大力发展/加快推进/重点支持）vs "限制类关键词"（规范/防止过热/严格监管）的出现频次变化趋势
- **数据来源**：`requests` + `BeautifulSoup` 爬取 `https://www.ndrc.gov.cn` 和 `http://www.xinhuanet.com`
- **展示方式**：词云 + 折线图（每周关键词频次趋势）
- **判断逻辑**：
  - 支持类关键词持续增加 → 政策持续加码（绿色）
  - 限制类关键词显著增加 → 政策转向预警（红色）

#### 指标 5.2：两类资产政策关注度对比
- **计算方法**：统计近30天政府官网文件中，"新能源/AI/科技/创新"相关词频 vs "地产/消费/传统制造"相关词频的比值
- **判断逻辑**：比值持续上升 → 政策向新经济倾斜不变；比值下降 → 政策开始平衡新旧经济（扭转预警）

## 综合信号：结构性牛市健康度评分

基于以上5个维度10个指标，计算一个综合得分（0~100分）：

| 维度 | 权重 | 满分 |
|------|------|------|
| 市场分化程度 | 30% | 30分 |
| 马太效应强度 | 20% | 20分 |
| 增量资金流向 | 25% | 25分 |
| 跨市场联动 | 15% | 15分 |
| 政策信号强度 | 10% | 10分 |

**评分区间与操作建议**：
- **75~100分**：结构性牛市特征强烈，维持"重仓新经济龙头"策略
- **50~74分**：结构性特征维持但有所弱化，适当提高防御仓位至20%
- **30~49分**：结构扭转信号明显，降低新经济持仓至50%，提高防御与现金至50%
- **0~29分**：结构性牛市可能终结，重新评估整体策略框架

## 预警系统设计

### 单指标预警（任一触发即提醒）

| 预警级别 | 触发条件 | 处理方式 |
|----------|----------|----------|
| 🟡 黄色预警 | 任意1个维度转为红色 | 控制台打印 + 日志记录 |
| 🟠 橙色预警 | 任意2个维度转为红色 | 打印 + 日志 + 生成预警报告 |
| 🔴 红色预警 | 3个及以上维度转为红色 | 打印 + 日志 + 报告 + 发送邮件通知 |

### 复合扭转信号（特别重要）

以下组合出现时，触发最高级预警：
1. 北向资金连续3周净流出 + 沪深300跑输中证1000 + 行业离散度降至25%分位以下
2. 政策限制类词频急升 + 地产ETF出现大规模净流入 + 新旧经济比值跌破0.9

## 技术实现要求

### 技术栈

```
Python 3.10+
数据获取：akshare（主要）、yfinance（美股数据）、requests + BeautifulSoup（政策爬虫）
数据处理：pandas、numpy、scipy
可视化：plotly（交互式图表）+ dash（Web仪表盘）
存储：SQLite（本地历史数据）
通知：smtplib（邮件）
调度：schedule（定时任务）
```

**页面一：总览仪表盘（首页）**

- 中部：5个维度的评分卡片（横排），每个卡片显示当前值、状态颜色、近期趋势小图
- 底部：最近30天健康度得分的折线图 + 预警事件时间轴

**页面二：分化指标详情**
- 申万31个行业近60日涨跌幅热力图（红绿对比）
- 新旧经济相对强度折线图（历史比值 + 阈值线）
- 涨跌家数比20日均线折线图

**页面三：资金流向详情**
- 北向资金20日净流入柱状图（正负双色）
- 北向资金行业分布饼图（最近10日流入方向）
- 主要行业ETF资金流入排行榜（表格）

**页面四：跨市场联动**
- 科创50 vs 纳斯达克100的60日滚动相关系数折线图
- 两个指数的同期走势对比图（双轴）
- 传导有效性（滞后1日相关系数）变化趋势

**页面五：政策信号**

**页面六：预警中心**
- 当前所有预警状态列表
- 历史预警记录（时间、级别、触发原因）
- 手动刷新按钮 + 最后更新时间

### 数据更新频率

| 数据类型 | 更新频率 | 说明 |
|----------|----------|------|
| 行情数据 | 交易日收盘后30分钟（15:30） | 非交易日跳过 |
| 北向资金 | 交易日每2小时 | 盘中可观察日内趋势 |
| ETF资金流 | 交易日收盘后 | 日度数据 |
| 美股数据 | 北京时间每天9:00（美股收盘后） | |
| 政策爬虫 | 每天7:00 | 抓取前一天新闻 |
| 综合评分 | 以上任一数据更新后自动重算 | |

### 错误处理要求

1. 数据获取失败时，使用最近一次成功获取的数据，并在界面显示"数据更新异常"提示
2. 某个指标计算失败不影响其他指标和综合评分（该指标用上次值代替）
3. 所有数据获取和计算过程记录日志到 `logs/` 目录
4. 数据库每周自动备份一次

## 初始化与运行说明

开发完成后，系统应支持以下运行方式：

```bash
# 首次初始化（建库、拉取历史数据）
python main.py --init

# 正常启动仪表盘（含定时更新）
python main.py

# 仅手动刷新数据（不启动Web界面）
python main.py --refresh

# 生成当日报告
python main.py --report
```

默认在 `http://localhost:8050` 运行Dash应用。

## 注意事项

1. **akshare限流**：部分接口有调用频率限制，数据获取时需加入适当延迟（`time.sleep(0.5)`）
2. **A股非交易日判断**：需要从akshare获取交易日历，非交易日不触发行情数据更新
3. **历史数据初始化**：首次运行时，需拉取至少2年历史数据用于计算历史分位阈值
4. **美股数据**：yfinance在中国大陆网络环境下可能需要代理，提供备选方案（akshare也有部分美股数据接口）
5. **政策爬虫合规**：爬取间隔不低于2秒，遵守robots.txt，仅爬取标题和摘要不爬取全文
6. **指标反向逻辑**：特别注意市场宽度（涨跌家数比）指标与结构性牛市的关系是**反向**的，需在代码注释和UI中清晰标注，避免误读
