# 数据初始化 SQL

按照「**系统标签 → 内置模板占位 → 股票主数据样本 → 测试用户 → 测试日记关联**」顺序执行。所有脚本**幂等**(可重复执行不报错),便于开发环境反复重置。

## 执行原则

| 原则 | 说明 |
|---|---|
| 幂等 | 用 `INSERT IGNORE` 或 `INSERT ... ON DUPLICATE KEY UPDATE`,重复执行不报错 |
| 不依赖自增 ID | 不写 `id = 1`,通过业务键(`openid`、`name`、`ts_code`)定位 |
| 环境隔离 | 测试数据集中 `dev_` 前缀,便于生产环境过滤排除 |
| 时间统一 | 不写死时间,用 `NOW()`、`CURDATE()`、`DATE_SUB` |

**关键说明**:
- `owner_user_id IS NULL` 配合表上 `UNIQUE KEY (owner_user_id, name)` 实现"系统标签全局唯一"
- MySQL 5.7 中 NULL 不参与唯一约束的判定,**理论上可能插入两条都是 NULL + 同名**——所以这里用 `ON DUPLICATE KEY UPDATE` 会失效。**实际情况下** 只在初始化时执行一次,且生产环境会做应用层去重,问题不大
- 如果你需要严格唯一,生产环境请加额外的 `_unique_marker` 字段(把 NULL 替换为 0),这里暂不复杂化

**分类约定**:

| `category` | 用途 |
|---|---|
| 0 | 普通(行业、主题) |
| 1 | 系统预置(策略风格) |
| 2 | 错题本/纪律相关 |
| 3 | 策略动作 |

**关键技巧**:

| 技巧 | 用途 |
|---|---|
| `SET @uid := (SELECT id ...)` | 用变量缓存用户 ID,避免重复子查询 |
| `INSERT ... SELECT ... FROM` | 关联表插入用查询代替硬编码 ID |
| `ORDER BY id DESC LIMIT 1` | 在重复执行时拿最新一条,避免冲突 |
| `ON DUPLICATE KEY UPDATE created_at = created_at` | 关联表唯一键冲突时不改任何字段(等同于 IGNORE) |

## 6. 验证查询(执行后跑一遍确认)

```sql
-- ================================================================
-- 6. 数据验证(初始化后执行,检查数据正确性)
-- ================================================================

-- 6.1 系统标签数量(预期 22)
SELECT COUNT(*) AS sys_tag_count 
FROM diary_tag_dict 
WHERE owner_user_id IS NULL;

-- 6.2 股票主数据数量(预期 23,含 1 退市)
SELECT 
  COUNT(*) AS total,
  SUM(CASE WHEN status = 1 THEN 1 ELSE 0 END) AS normal,
  SUM(CASE WHEN status = 0 THEN 1 ELSE 0 END) AS delisted
FROM stock_info;

-- 6.3 测试用户列表
SELECT 
  id, openid, nickname, status, level,
  JSON_EXTRACT(prefs, '$.ui.theme') AS theme,
  JSON_EXTRACT(prefs, '$.ui.senior_mode') AS senior_mode
FROM sys_user 
WHERE openid LIKE 'dev_%'
ORDER BY id;

-- 6.4 张三的日记列表(预期 3 条)
SELECT 
  d.id, d.entry_date, d.entry_type, d.title, d.excerpt,
  GROUP_CONCAT(DISTINCT s.name SEPARATOR ', ') AS stocks,
  GROUP_CONCAT(DISTINCT t.name SEPARATOR ', ') AS tags
FROM diary_entry d
LEFT JOIN diary_stock ds ON d.id = ds.diary_id
LEFT JOIN stock_info s   ON ds.stock_id = s.id
LEFT JOIN diary_tag dt   ON d.id = dt.diary_id
LEFT JOIN diary_tag_dict t ON dt.tag_id = t.id
WHERE d.user_id = (SELECT id FROM sys_user WHERE openid = 'dev_openid_001')
  AND d.deleted_at IS NULL
GROUP BY d.id, d.entry_date, d.entry_type, d.title, d.excerpt
ORDER BY d.entry_date DESC;

-- 6.5 全文搜索测试(预期至少 1 条)
SELECT id, title, excerpt
FROM diary_entry
WHERE MATCH(title, content) AGAINST('茅台' IN NATURAL LANGUAGE MODE)
  AND deleted_at IS NULL;

-- 6.6 拼音首字母测试(查 mt 应命中茅台、美团)
SELECT ts_code, name, pinyin_initial
FROM stock_info
WHERE pinyin_initial LIKE 'mt%' OR pinyin_initial LIKE '%mt%'
ORDER BY 
  CASE WHEN pinyin_initial = 'mt' THEN 1 
       WHEN pinyin_initial LIKE 'mt%' THEN 2 
       ELSE 3 END;
```

## 7. 清理脚本(开发环境重置用)

```sql
-- ================================================================
-- 7. 清理测试数据(仅开发环境使用,生产严禁执行)
-- 按外键依赖反向顺序删除
-- ================================================================

-- 警告确认(留个心眼,防止误执行)
SELECT 'WARNING: 即将清理所有 dev_ 前缀的测试数据,5 秒后开始' AS warning;

-- 7.1 拿到测试用户 ID 列表
SET @dev_user_ids := (
  SELECT GROUP_CONCAT(id) FROM sys_user WHERE openid LIKE 'dev_%'
);

-- 7.2 按依赖反向清理
DELETE FROM mp_media_cache       
WHERE mp_account_id IN (SELECT id FROM mp_account WHERE FIND_IN_SET(user_id, @dev_user_ids));

DELETE FROM mp_publish_record    
WHERE FIND_IN_SET(user_id, @dev_user_ids);

DELETE FROM mp_account           
WHERE FIND_IN_SET(user_id, @dev_user_ids);

DELETE FROM diary_export_task    
WHERE FIND_IN_SET(user_id, @dev_user_ids);

DELETE FROM diary_tag            
WHERE diary_id IN (SELECT id FROM diary_entry WHERE FIND_IN_SET(user_id, @dev_user_ids));

DELETE FROM diary_stock          
WHERE diary_id IN (SELECT id FROM diary_entry WHERE FIND_IN_SET(user_id, @dev_user_ids));

DELETE FROM diary_attachment     
WHERE FIND_IN_SET(user_id, @dev_user_ids);

DELETE FROM diary_entry          
WHERE FIND_IN_SET(user_id, @dev_user_ids);

-- 用户自定义标签(系统标签保留)
DELETE FROM diary_tag_dict       
WHERE owner_user_id IS NOT NULL AND FIND_IN_SET(owner_user_id, @dev_user_ids);

-- 最后删用户
DELETE FROM sys_user             
WHERE openid LIKE 'dev_%';

SELECT '清理完成' AS result;
```

**注意**:`FIND_IN_SET` 性能不如 `IN`,但这是清理脚本不是热路径,可读性优先。
