## E2 · 信号判定与标签体系

### E2-S1 派生指标计算

派生指标层包含量能、涨跌幅、排名及均线形态等 20 余项核心指标,每日优先计算。

#### E2-S1-T1 计算逻辑 (Python)
使用 pandas 进行滑动窗口计算,处理 MySQL 5.7 缺失的窗口函数功能。
- **量量比**: 计算 5 日/20 日量比均值。
- **行业分位**: 计算个股在申万一级行业内的当日涨幅分位。
- **压力位测试**: 识别 60 日内压力位并统计测试次数。

### E2-S2 标签判定体系

v1.1 引入 7 大类、约 80 个标签,形成"弹性"描述能力。

#### E2-S2-T1 标签类别与数据源
| 类别 | 判定方式 | 数据源 | 示例标签 |
|---|---|---|---|
| 价格行为 | 脚本直接判定 | K 线 | zt, breakout_60d |
| 资金 | SQL 判定 | L3/龙虎榜 | main_inflow_strong |
| 板块 | SQL + 派生指标 | L2 | sector_leader |
| 催化 | SQL JOIN | L6 | announcement_today |
| 历史 | 派生指标映射 | 派生指标表 | high_position |
| 形态 | Python 逻辑 | K 线/派生指标 | pre_zt_consolidation |
| 异常 | SQL 判定 | 基础信息 | st, micro_cap |

#### E2-S2-T2 互斥规则与冲突裁决
利用 `dim_tag_relation` 管理标签间的互斥关系(如"缩量"与"放量"互斥)。系统在计算完成后根据预设优先级保留最高价值标签。

### E2-S3 异动池产出 (三池统一)

#### E2-S3-T1 strong 池 (强异动)
- **同步**: 从 L8 同步所有强异动。
- **关键变化**: 不在 SQL 阶段硬过滤 ST/新股,而是全部入库并标注 `excluded_reasons` 及 `default_visible` 状态。

#### E2-S3-T2 early 池 (启动前)
包含 4 类高价值组合信号:
1.  **组合 1 龙头预备役**: 排名跃升 + 量能温和放大。
2.  **组合 3 连板接力候选**: 板块首板数爆发 + 个股强力流入。
3.  **组合 2 箱体突破蓄势**: 均线粘合 + 压力位多次测试。
4.  **组合 4 趋势反转早期**: 首次站稳年线 + 缩量回踩。

#### E2-S3-T3 trap 池 (陷阱信号)
识别诱多放量、假突破、高位出货及领涨股见顶等 4 类陷阱,用于避错训练。

### E2-S4 多维印证计算

#### E2-S4-T1 共振等级 (Resonance Level)
基于 6 大维度 (自身、资金、板块、大势、外部、时间) 的命中情况,计算 L1-L5 共振等级。
- **L5 (最高)**: 5 个及以上维度出现正向印证。

#### E2-S4-T2 反向信号与抑制
自动识别基本面利空 (减持、解禁) 及情绪过热信号。反向信号会计算 `counter_signal_score`,用于在评分阶段进行抑制。
