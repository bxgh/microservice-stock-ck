# L8 异动池 v1 极简版 · 设计全景 (Overview)

> **版本**: v1.0 (极简版)
> **日期**: 2026-05-07
> **设计哲学**: 鲁棒性 > 拟合度, 评估系统先于评分系统建立。

---

## 1. 背景与目标

### 1.1 当前痛点
- 评分系统没有评估能力, 无法判断改动是否有效。
- 评分分布偏态, 权重缺乏依据。
- 极端市况(普涨/普跌)下系统失效。

### 1.2 设计核心
本项目放弃复杂的 v1.1 方案(22 个标签、共振等级等), 转而建立**极简版骨架**:
1. **评估子系统**: 先建立回测机制, 用数据指导评分优化。
2. **异动分类**: 引入 C1-C4 机制分类, 为分组诊断打基础。
3. **极端市况门控**: 识别普涨/普跌, 自动切换到 D 视图(全景简报)。

---

## 2. 核心目标
- **业务**: 建立月度回测报告机制; 极端市况下系统不失语。
- **技术**: 增量改造主表 schema; 新增 2 张评估表; 100% 任务接入自动化调度。

## 6. 风险与缓解
- **数据缺失**: 允许 NULL 样本, 报告中记录剔除率。
- **日期跳跃**: 强制使用 `meta_trading_calendar` 链路, 严禁日历日加减。
- **性能**: 月度回测错峰执行(02:00)。
