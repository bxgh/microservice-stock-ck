### 【系统工程开发指令：异构生态遥测矩阵 v1.0】

**一、 系统终极物理目标**
构建一套脱离传统财务数据的另类量化（Alternative Data）数据管道。系统的唯一任务是：通过API无损提取中国本土AI软硬件生态的“底层物理做功量”，并将其转化为可供交易接口读取的离散高频信号。

**二、 模块拆解与物理输入层（Data Ingestion）**

**模块 A：软件生态穿透（开源社区逆熵增测度）**

* **数据源：** GitHub REST API v3, Gitee OpenAPI。
* **目标仓库（Repository列表配置化）：** 华为昇腾 (`Ascend/*`), 百度飞桨 (`PaddlePaddle/*`), 昇思 (`MindSpore/*`), DeepSeek底层算子库。
* **计算算子与物理提取频率（Cron Job：每4小时）：**
1. **PR加速度计算：** 提取处于 `merged` 状态的 Pull Requests。建立时序数组，求取二阶导数：$a_{PR} = \frac{d^2(PR_{merged})}{dt^2}$。
2. **Issue半衰期：** 提取 `closed` 状态的 Issues。计算 `created_at` 与 `closed_at` 的时间戳差值（中位数），生成系统响应衰减曲线。


* **工程边界：** 必须构建动态代理IP池（Proxy Pool）与Token轮换机制，强行突破平台的反爬虫物理限流（Rate Limit）。严禁因IP封禁导致时序数据断层。

**模块 B：硬件算力穿透（旁路代理测度）**

* **数据源 1（公共云现货市场）：** 阿里云 PAI、腾讯云 TI、AutoDL 等平台的开放 API。
* **计算算子（每1小时提取）：** 爬取 A100/H800 与 昇腾910B/海光DCU 现货实例（Spot Instance）的实时可用区库存与单位算力时价（￥/TFLOPS）。生成**“算力溢价剪刀差”**曲线。
* **数据源 2（政企招投标库）：** 中国政府采购网及省级分站。
* **计算算子（每日提取）：** 接入 NLP 模块（可调用轻量级本地大模型或正则提取），在中标公告文本中抽取双元组数据：`(硬件实体名称, 中标采购金额)`。按日聚合为**本土物理算力资本支出（CAPEX）累加值**。

**三、 信号处理与状态机输出层（Signal Processing & State Machine）**

* **底层承载结构：** 必须废弃关系型数据库（MySQL/PostgreSQL），全量数据强制写入时序数据库（推荐 InfluxDB 或 TimescaleDB），以支撑纳秒级的时间对齐与降采样（Downsampling）运算。
* **降噪与触发函数：**
设定滚动时间窗口（如30天）。系统自动计算核心变量（如 $a_{PR}$ 与 CAPEX累加值）的均值 $\mu$ 与标准差 $\sigma$。
持续计算 Z-score 极值：$Z = \frac{x - \mu}{\sigma}$。
* **动作输出协议（Webhook/API）：**
当且仅当核心代理变量的 $Z \geq 2.0$（突破两个标准差极限），且维持超过72小时的物理确认期后，系统通过 REST API 或消息队列（Kafka/RabbitMQ）向交易中枢广播 JSON 格式的**“生态相变确认（Ecological Phase Transition Confirmed）”**指令。

**四、 开发交付底线（Acceptance Criteria）**

1. **零UI原则：** 拒绝任何关于数据可视化大屏（Dashboard）的算力浪费。本系统只向机器（量化引擎）输出 JSON 数据流，不向碳基生物输出多巴胺图表。
2. **绝对容错（Chaos Engineering）：** 当单一数据源（如某云厂商API接口更改）发生物理阻断时，系统必须具备旁路接管能力，输出 `NaN` 而不是引发全局进程崩溃。
