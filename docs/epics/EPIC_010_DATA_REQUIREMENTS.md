## 1. 数据源概览

### 1.1 外部数据源

| 数据源 | 类型 | 用途 | 当前状态 |
|--------|------|------|----------|
| 腾讯云 MySQL | 关系型数据库 | 财务报表、历史数据主库 | ✅ 已开通 |
| Baostock API | REST API | 历史K线、财务指标 | ✅ 可用 |
| AkShare API | REST API | 实时行情、指数成分 | ✅ 可用 |
| Mootdx | TCP | Level2 行情、分笔数据 | ✅ 可用 |

### 1.2 本地存储

| 存储 | 类型 | 用途 | 当前状态 |
|------|------|------|----------|
| ClickHouse | 时序数据库 | 高性能时序数据存储 | ✅ 已部署 |
| Redis | 缓存 | 热数据缓存 | ✅ 已部署 |
| PostgreSQL | 关系型数据库 | 元数据、配置数据 | ✅ 已部署 |

### 1.3 数据源选型论证

#### 腾讯云 MySQL

| 维度 | 论证 |
|------|------|
| **选择理由** | 已有历史数据沉淀；团队熟悉运维；云端高可用 |
| **替代方案** | 自建 MySQL / RDS |
| **不选替代的原因** | 数据迁移成本高；现有数据已在腾讯云 |
| **风险** | 网络延迟 (50-100ms)；云端故障导致不可用 |
| **缓解措施** | 本地 ClickHouse 缓存；定时同步降低实时依赖 |

#### Baostock / AkShare

| 维度 | 论证 |
|------|------|
| **选择理由** | 免费开源；社区活跃；覆盖 A 股全市场 |
| **替代方案** | Tushare Pro (付费)、Wind (商用) |
| **不选替代的原因** | 成本考虑；当前数据需求可满足 |
| **风险** | 限流、数据延迟、接口变更 |
| **缓解措施** | 多数据源冗余；熔断重试机制 |

#### Mootdx

| 维度 | 论证 |
|------|------|
| **选择理由** | Level2 数据支持；分笔数据获取；低延迟 |
| **替代方案** | 券商 API、聚宽本地 |
| **不选替代的原因** | 券商 API 需开户；聚宽需付费 |
| **风险** | 非官方接口；稳定性依赖通达信服务器 |
| **缓解措施** | 连接池复用；多服务器轮询 |

#### ClickHouse (本地存储)

| 维度 | 论证 |
|------|------|
| **选择理由** | 列式存储高压缩比；时序查询毫秒级；已部署 |
| **替代方案** | TimescaleDB、InfluxDB |
| **不选替代的原因** | ClickHouse 查询性能更优；团队已有使用经验 |
| **适用场景** | 日K线、分笔数据、因子值等时序数据 |

## 2. 数据表设计

### 2.1 日K线数据表 (ClickHouse)

**表名**: `kline_daily`

| 字段名 | 类型 | 说明 | 索引 |
|--------|------|------|------|
| `stock_code` | String | 股票代码 | ✅ 主键 |
| `trade_date` | Date | 交易日期 | ✅ 主键 |
| `open` | Float64 | 开盘价 | |
| `high` | Float64 | 最高价 | |
| `low` | Float64 | 最低价 | |
| `close` | Float64 | 收盘价 | |
| `volume` | UInt64 | 成交量 (股) | |
| `amount` | Float64 | 成交额 (元) | |
| `turnover_rate` | Float32 | 换手率 (%) | |
| `adj_factor` | Float64 | 复权因子 | |

**分区策略**: 按 `toYYYYMM(trade_date)` 月度分区  
**数据保留**: 永久保留

## 3. 数据同步需求

### 3.1 腾讯云 MySQL → 本地 ClickHouse

| 同步任务 | 源表 | 目标表 | 频率 | 触发时间 |
|----------|------|--------|------|----------|
| 财务数据同步 | `financials` | `financials` | 季度 | 财报发布后 3 天 |
| 历史K线补录 | `kline_history` | `kline_daily` | 按需 | 手动触发 |

### 3.2 外部 API → 本地 ClickHouse

| 同步任务 | 数据源 | 目标表 | 频率 | 触发时间 |
|----------|--------|--------|------|----------|
| 日K线更新 | Baostock | `kline_daily` | 每日 | 18:00 CST |
| 估值更新 | AkShare | `valuation_history` | 每日 | 18:30 CST |
| 分笔数据 | Mootdx | `tick_data` | 实时 | 交易时间 |

## 4. 数据质量标准

| 维度 | 标准 | 检查方式 |
|------|------|----------|
| 完整性 | 日K线无缺失交易日 | 对比交易日历 |
| 准确性 | 价格偏差 < 0.01% | 与官方数据比对 |
| 时效性 | 盘后数据 T+0 可用 | 监控同步延迟 |
| 一致性 | 云端与本地一致 | 每周对账任务 |
