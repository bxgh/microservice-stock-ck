## 4. 用户故事

### Story 14.1: 自愈机制验证
**优先级**: P0  
验证 Weekly Deep Audit 触发自愈的完整流程。

**验收标准**:
- [ ] 手动制造数据不一致
- [ ] 验证 audit 脚本检测到不一致
- [ ] 验证自动删除分区并重新同步
- [ ] 验证最终数据一致

### Story 14.2: 全链路压测
**优先级**: P0  
执行 500 QPS 并发压测，模拟多策略同时查询。

**验收标准**:
- [ ] 压测工具 (Locust/wrk) 配置
- [ ] 持续 10 分钟无错误
- [ ] P95 延迟 < 100ms
- [ ] CPU/内存使用率 < 80%

### Story 14.3: 双主故障演练
**优先级**: P0  
模拟单节点故障，验证另一节点继续服务。

**验收标准**:
- [ ] 停止 Server 41，验证 Server 58 可读写
- [ ] 恢复 Server 41，验证数据自动同步
- [ ] 记录恢复时间和步骤

### Story 14.4: ClickHouse 调优
**优先级**: P1  
根据压测结果调优 ClickHouse 参数。

**验收标准**:
- [ ] 分区策略优化
- [ ] TTL 配置验证 (180 天)
- [ ] 批写参数调整
- [ ] 调优配置文档记录

### Story 14.5: 运维手册编写
**优先级**: P1  
编写完整运维手册，覆盖日常操作。

**验收标准**:
- [ ] 服务启停流程
- [ ] 日志查看方法
- [ ] 常用 SQL 查询
- [ ] 监控告警响应

### Story 14.6: 故障恢复流程文档
**优先级**: P1  
编写故障恢复标准流程。

**验收标准**:
- [ ] ClickHouse 单节点故障恢复
- [ ] SSH 隧道断开恢复
- [ ] 数据不一致修复
- [ ] 服务无响应排查

### Story 14.7: 数据本地化方案文档
**优先级**: P2  
完善《数据本地化方案》文档。

**验收标准**:
- [ ] 架构全景图
- [ ] 数据流详细描述
- [ ] 一致性保障机制
- [ ] 运维指南
