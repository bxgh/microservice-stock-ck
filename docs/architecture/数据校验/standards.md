# 数据校验标准手册 (Data Validation Standards)

本文档定义了系统中各类金融核心数据的校验规则。所有规则在 `libs/gsd-shared` 中统一实现。

## 1. 分笔数据 (Tick Data)

分笔数据根据生命周期的不同，应用四个层级的校验标准。

| 校验维度 | 1. 采集过滤 (Loose) | 2. 盘中监控 (IntradayRealtime) | 3. 盘后审计 (IntradayPostMarket) | 4. 历史归档 (History) |
| :--- | :--- | :--- | :--- | :--- |
| **应用场景** | 采集任务前置检查 (Redis/Local) | Gate-1 实时监控 | Gate-3 当日数据审计 | 数据一致性巡检 / 模型训练 |
| **关注点** | 存在性 (防止重复采集) | 时效性 (新鲜度) | 完整性 (容忍少许波动) | 准确性 (严谨无误) |
| **时间范围** | 10:00 - 14:30 有数据即可 | N/A | 09:25:00 - 15:00:00 | 09:25:05 - 14:59:55 |
| **数据量(min)**| >= 2000 ticks | N/A | >= 230 Active Minutes | >= 237 Active Minutes |
| **最大延迟** | N/A | < 180 秒 | N/A | N/A |
| **收盘价对账**| N/A | N/A | Diff < 2.0% | Diff < 1.1% |
| **成交量对账**| N/A | N/A | Diff < 10.0% | Diff < 5.0% |

> **注**: 对账基准为交易所发布的日 K 线数据。

## 2. K线数据 (KLine Data)

K线数据作为系统的基准数据，要求极高的准确性。

| 规则名称 | 阈值/标准 | 说明 |
| :--- | :--- | :--- |
| **覆盖率** | >= 98.0% | 当日采集数 / 市场实际交易股票数 |
| **连续性** | Max Gap <= 10 days | 排除停牌后的最大允许交易日缺口 |
| **OHLC逻辑** | High >= Low/Open/Close <br> Low <= Open/Close | 基础价格逻辑检查 |
| **异常波动** | PctChange <= 20% | 过滤明显错误的价格数据 (如除权未复权导致) |

## 3. 股票名单 (Stock List)

每日同步的股票基础信息校验。

| 规则名称 | 阈值/标准 | 说明 |
| :--- | :--- | :--- |
| **数量范围** | 4500 - 6000 | 防止名单被意外清空或注入垃圾数据 |
| **变动比例** | <= 2% | 每日新增/退市比例不应过大 |
| **重叠率** | >= 95% | 与昨日名单的重叠度，防止名单剧变 |
| **代码格式** | 符合交易所规则 | 60/68/00/30 开头，6位数字 |

## 4. 全市场级 (Market Level)

在 Gate-3 审计中对市场整体状态的检查。

| 规则名称 | 阈值 | 说明 |
| :--- | :--- | :--- |
| **K线覆盖率** | >= 98.0% | 全市场维度 |
| **Tick覆盖率** | >= 95.0% | 有Tick股票数 / 有K线股票数 |
| **异常总数** | <= 500 只 | 连续性检查不合格的股票总数上限 |

## 5. 规则代码映射

所有阈值由代码中的常量定义，修改需经过 Code Review。

- **代码位置**: `libs/gsd-shared/gsd_shared/validation/standards.py`
- **涉及服务**:
    - `gsd-worker` (TickSync, PostMarketGate)
    - `task-orchestrator` (DataGates)
