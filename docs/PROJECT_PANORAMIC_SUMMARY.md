# 项目全景总结：功能、范围与作用

本项目是一个**企业级 A 股量化数据处理与任务编排系统**，旨在为高频/中频量化策提供高可靠、低延迟的数据底座与自动化执行环境。

## 1. 核心功能 (Functionality)

### A. 全自动化数据生命周期管理
*   **多源同步**：集成 Mootdx、AkShare、BaoStock 等数据源，实现 L2 级分笔数据（Tick）、K 线、财务数据及龙虎榜数据的全量/增量采集。
*   **三级质量门禁 (Data Gates)**：
    - **Gate-1 (盘前)**：环境与基准数据就绪检查。
    - **Gate-2 (盘中)**：实时采集覆盖率监控与断线自愈。
    - **Gate-3 (盘后)**：基于快照真值的全量审计、异常剔除与自动化重试补采。

### B. 分布式任务编排与治理
*   **双链路调度**：采用 APScheduler + MySQL 指令库 + Worker 轮询架构，实现调度与执行的物理隔离。
*   **Workflow 4.0**：支持 DAG 依赖解析、Handlebars 参数注入及基于 AI (gsd-agent) 的故障自动诊断与恢复。

### C. 深度量化因子计算
*   **特征工程**：基于分笔数据生成 OFI (订单流失衡)、Smart Money (聪明钱流动) 等 9 维核心特征矩阵。
*   **聚类分析**：利用 DTW (动态时间规整) 进行行为指纹聚类，识别板块龙头及领涨滞后关系。

---

## 2. 项目范围 (Scope)

*   **市场范围**: 聚焦于 A 股全市场（沪深京），包含 5400+ A 股、北证个股，以及 200+ 活跃 ETF 与核心基准指数。
*   **数据粒度**：涵盖从 3 秒一跳的 L2 分笔数据到日线级别的全频率数据。
*   **架构边界**：
    - **业务层**：量化策略逻辑与信号生成。
    - **控制层**：分布式任务调度、门禁审计、AI 运维。
    - **数据层**：ClickHouse (海量历史数据)、Redis (实时状态缓存)、MySQL (配置与指令集)。

---

## 3. 各模块作用与角色 (Roles)

| 模块名称 | 核心角色 | 主要职责 |
| :--- | :--- | :--- |
| **task-orchestrator** | **系统大脑 (Master)** | 发射任务指令、管理 Workflow 进度、驱动 AI 诊断。 |
| **gsd-worker** | **执行主力 (Worker)** | 执行具体同步作业、运行审计逻辑、进行数据归档。 |
| **get-stockdata** | **数据网关 (API)** | 统一暴露行情访问接口，屏蔽底多个异构数据源的差异。 |
| **mootdx-source/api** | **实时源适配器** | 负责与通达信服务器建立物理连接，维护连接池。 |
| **quant-strategy** | **计算引擎** | 特征提取、向量化计算、聚类算法运行。 |
| **gsd-shared (Libs)** | **标准库** | 定义全局统一的代码规范、数据模型与工具类。 |
| **gsd-agent (Libs)** | **大脑外挂** | 为系统提供 LLM 智能推理能力，用于日志诊断与策略生成。 |

---

## 4. 总结 (Summary)

该项目不仅是一个爬虫系统，更是一个具备**自愈能力 (Self-healing)** 的生成式 AI + 传统金融工程结合的闭环架构。它通过“指令驱动”解决了大规模集群的运维难题，通过“三级门禁”保障了金融级的数据完整性，为量化投研提供了极高价值的“数据净水厂”环境。
