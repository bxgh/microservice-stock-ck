# A股细分数据类型精确接口使用策略

## 概述

本文档针对A股市场的6大核心数据类型，制定了精确的接口使用策略，通过专业分工确保每种数据都使用最适合的接口，最大化数据质量和采集效率。

### 数据类型分类体系

1. **实时行情数据** - 毫秒级更新，交易决策支持
2. **历史K线数据** - 分钟级到年级，技术分析基础
3. **财务数据** - 季度/年度更新，价值投资核心
4. **公司基本面数据** - 公司画像，投资研究基础
5. **市场情绪和资金流向数据** - 市场温度计，情绪指标
6. **交易数据和行为分析** - 交易行为洞察，机会发现

## 第一部分：实时行情数据接口策略

### 1.1 实时价格数据采集策略

#### 主接口：新浪财经API
**接口地址**: `https://hq.sinajs.cn/list=`
**专长分析**:
- **实时性**: 1-3秒更新，市场最快
- **覆盖面**: 沪深京A股全覆盖
- **稳定性**: 长期稳定，极少中断
- **数据格式**: 简洁易解析，处理效率高

**采集策略**:
```
更新频率: 每3秒一次
优先级: P0 (最高优先级)
调度方式: 实时推送模式
数据字段:
- 当前价、昨收价、开盘价
- 最高价、最低价、成交量
- 成交额、更新时间
```

#### 备选接口：腾讯财经API
**接口地址**: `https://qt.gtimg.cn/q=`
**专长分析**:
- **数据丰富**: 字段比新浪更详细
- **格式统一**: 使用~分隔符，解析简单
- **稳定性**: 服务器稳定，响应快速
- **扩展性**: 支持更多股票信息

**使用场景**:
- 新浪财经数据验证
- 详细数据补充获取
- 主接口故障时的备选方案

#### 数据质量控制
```
验证规则:
1. 价格连续性检查 (跳跃>5%重新获取)
2. 成交量合理性验证 (不能为负)
3. 更新时间连续性检查
4. 多源数据对比验证
```

### 2.4 复权数据处理策略

#### 专业复权处理：baostock
**复权类型**:
```
前复权 (qfq):
- 保持当前价格不变
- 历史价格向前调整
- 适合技术分析、趋势判断

后复权 (hfq):
- 保持历史价格不变
- 当前价格向后调整
- 适合收益率计算、对比分析

不复权 (dq):
- 保持原始价格不变
- 显示真实价格走势
- 适合成本分析、实际收益
```

#### 复权因子计算
```
复权因子 = (当日收盘价 + 当日分红) / 次日开盘价
复权价格 = 原始价格 × 复权因子序列累乘
```
