# 分笔数据获取脚本使用说明

## 概述

本脚本基于 `mootdx` 库实现股票分笔成交数据的获取功能。分笔数据（Tick Data）记录了每一笔成交的详细信息，包括成交时间、价格、数量、金额和买卖方向等。

## 文件说明

- `tick_data_fetcher.py` - 主要的分笔数据获取脚本
- `分笔数据使用说明.md` - 本文档

## 依赖要求

确保已安装以下依赖：
```bash
pip install mootdx[all] pandas
```

## 核心类说明

### TickDataFetcher 类

主要的分笔数据获取器，提供以下功能：

#### 初始化参数
- `market`: 市场类型，默认 'std'（标准市场）
- `server`: 服务器地址和端口，默认 ('60.191.117.167', 7709)

#### 主要方法

1. **`get_transactions_data(symbol, date, start=0, offset=1000)`**
   - 获取单个股票的分笔成交数据
   - 参数:
     - `symbol`: 股票代码，如 '000001'
     - `date`: 交易日期，格式 'YYYYMMDD'
     - `start`: 起始位置，默认0
     - `offset`: 获取数量，默认1000

2. **`get_multiple_stocks_transactions(symbols, date, start=0, offset=500)`**
   - 批量获取多个股票的分笔数据
   - 参数:
     - `symbols`: 股票代码列表
     - `date`: 交易日期
     - `start`: 起始位置
     - `offset`: 每个股票获取数量

3. **`save_to_csv(data, filename=None)`**
   - 将分笔数据保存到CSV文件
   - 支持单个DataFrame和多个股票的字典数据

4. **`display_data_info(data)`**
   - 显示分笔数据的基本信息

5. **`close()`**
   - 关闭客户端连接

## 使用示例

### 基本使用

```python
from tick_data_fetcher import TickDataFetcher

# 创建获取器
fetcher = TickDataFetcher()

# 获取单个股票分笔数据
data = fetcher.get_transactions_data(
    symbol='000001',    # 平安银行
    date='20241028',    # 指定日期
    offset=100          # 获取100条记录
)

# 显示数据信息
fetcher.display_data_info(data)

# 保存到CSV
fetcher.save_to_csv(data, 'tick_data.csv')

# 关闭连接
fetcher.close()
```

### 批量获取

```python
# 批量获取多个股票
symbols = ['000001', '600000', '000858']
batch_data = fetcher.get_multiple_stocks_transactions(
    symbols=symbols,
    date='20241028',
    offset=50
)

# 显示批量数据信息
fetcher.display_data_info(batch_data)

# 保存批量数据
fetcher.save_to_csv(batch_data, 'batch_tick_data.csv')
```

### 直接运行脚本

```bash
cd debug
python tick_data_fetcher.py
```

脚本会自动运行演示示例，包括：
1. 获取单个股票分笔数据
2. 批量获取多个股票分笔数据
3. 保存数据到CSV文件

## 数据字段说明

分笔数据包含以下主要字段：

| 字段名 | 说明 | 示例 |
|--------|------|------|
| `time` | 成交时间 | 14:56 |
| `price` | 成交价格 | 11.63 |
| `vol` | 成交数量（股） | 167 |
| `buyorsell` | 买卖方向 | 0（未知）/1（卖出）/2（买入） |
| `volume` | 成交量（可能与vol相同） | 167 |

**买卖方向说明：**
- `0`: 未知方向
- `1`: 主动卖出（内盘）
- `2`: 主动买入（外盘）

## 注意事项

1. **数据时效性**
   - 分笔数据通常只保留最近一段时间
   - 历史数据可能无法获取

2. **数据量限制**
   - 单次请求建议不超过1000条记录
   - 批量获取时注意网络负载

3. **网络连接**
   - 需要稳定的通达信服务器连接
   - 建议添加重试机制

4. **错误处理**
   - 脚本已包含基本的异常处理
   - 建议在生产环境中添加更完善的错误处理

5. **数据格式**
   - 返回数据为pandas DataFrame格式
   - 保存为CSV时使用utf-8-sig编码支持中文

## 性能优化建议

1. **连接复用**
   - 避免频繁创建和关闭连接
   - 使用连接池管理

2. **批量操作**
   - 尽量使用批量获取方法
   - 合理设置offset参数

3. **异步处理**
   - 对于大量股票，考虑使用异步请求

4. **数据缓存**
   - 对重复请求的数据进行缓存

## 常见问题

### Q: 获取不到数据怎么办？
A:
1. 检查网络连接是否正常
2. 确认股票代码和日期格式正确
3. 尝试不同的服务器地址
4. 检查mootdx库版本

### Q: 数据字段不全怎么办？
A:
1. 检查mootdx库版本是否支持完整字段
2. 查看官方文档了解字段映射
3. 可能需要升级mootdx库

### Q: 如何获取实时分笔数据？
A:
1. 使用WebSocket连接实时数据
2. 设置定时任务定期获取
3. 结合其他实时数据源
