## 1. 全市场股票池 (Universe Pool) 数据需求

### 1.1 全市场股票列表
**接口**: `GET /api/v1/stocks/list`

**请求参数**:
```json
{
  "limit": 5000,
  "exchange": "ALL"  // 可选: SH, SZ, BJ
}
```

**必需字段**:
| 字段名 | 类型 | 说明 | 示例 |
|-------|------|------|------|
| `code` | String | 股票代码 (6位) | "600519" |
| `name` | String | 股票名称 | "贵州茅台" |
| `exchange` | String | 交易所 | "SH", "SZ", "BJ" |
| `list_date` | Date | 上市日期 | "2001-08-27" |
| `market_cap` | Float | 总市值 (亿元) | 2500.5 |
| `avg_turnover_20d` | Float | 20日日均成交额 (万元) | 150000.0 |
| `turnover_ratio_20d` | Float | 20日平均换手率 (%) | 1.2 |

**可选字段** (用于高级筛选):
- `industry`: 行业分类 (申万一级)
- `pe_ttm`: 市盈率 (TTM)
- `pb`: 市净率
- `total_shares`: 总股本 (万股)

**数据更新频率**: 每日盘后更新

## 2. 候选池 (Candidate Pool) 数据需求

### 2.1 实时行情数据
**接口**: `GET /api/v1/quotes/realtime` (批量)

**请求参数**:
```json
{
  "codes": ["600519", "000001", "000002"]  // 最多 300 只
}
```

**必需字段**:
| 字段名 | 类型 | 说明 | 示例 |
|-------|------|------|------|
| `code` | String | 股票代码 | "600519" |
| `name` | String | 股票名称 | "贵州茅台" |
| `price` | Float | 最新价 | 1850.50 |
| `volume` | Integer | 成交量 (股) | 1234567 |
| `turnover` | Float | 成交额 (元) | 2280000000.0 |
| `change_pct` | Float | 涨跌幅 (%) | 2.35 |
| `timestamp` | DateTime | 行情时间 | "2025-12-14T15:00:00+08:00" |

**可选字段** (用于资金流分析):
- `buy_volume`: 主动买入量
- `sell_volume`: 主动卖出量
- `big_order_in`: 大单流入 (万元)
- `big_order_out`: 大单流出 (万元)

**数据更新频率**: 实时 (交易时间内每 3 秒更新)

### 2.2 财务指标数据 (用于长线候选池分类)
**接口**: `GET /api/v1/stocks/{code}/fundamentals`

**必需字段**:
| 字段名 | 类型 | 说明 | 用途 |
|-------|------|------|------|
| `dividend_yield` | Float | 股息率 (%) | 红利池分类 |
| `roe` | Float | 净资产收益率 (%) | 成长池分类 |
| `revenue_growth_yoy` | Float | 营收同比增长率 (%) | 成长池分类 |
| `net_profit_margin` | Float | 净利润率 (%) | 质量评估 |

**数据更新频率**: 每季度财报发布后更新

## 3. 持仓池 (Position Pool) 数据需求

### 3.1 流动性数据 (关键!)
**接口**: `GET /api/v1/stocks/{code}/liquidity`

**必需字段**:
| 字段名 | 类型 | 说明 | 用途 |
|-------|------|------|------|
| `avg_daily_volume` | Float | 20日日均成交额 (元) | 流动性检查 |
| `bid_ask_spread` | Float | 买卖价差 (元) | 冲击成本估算 |
| `order_book_depth_5` | Object | 5档盘口深度 | 大单冲击评估 |

**order_book_depth_5 结构**:
```json
{
  "bids": [
    {"price": 1850.00, "volume": 1000},
    {"price": 1849.50, "volume": 2000},
    ...
  ],
  "asks": [
    {"price": 1850.50, "volume": 1500},
    {"price": 1851.00, "volume": 2500},
    ...
  ]
}
```

**数据更新频率**: 实时 (交易时间内)

### 3.2 持仓盈亏计算用数据
**接口**: 复用 `GET /api/v1/quotes/realtime`

**需求**: 
- 支持批量查询 (最多 100 只持仓股票)
- 返回 `price` 字段用于计算浮动盈亏

## 4. 黑名单池 (Blacklist Pool) 数据需求

### 4.1 股票状态监控
**接口**: `GET /api/v1/stocks/{code}/status`

**必需字段**:
| 字段名 | 类型 | 说明 | 用途 |
|-------|------|------|------|
| `is_st` | Boolean | 是否 ST 股票 | 自动加入黑名单 |
| `is_suspended` | Boolean | 是否停牌 | 流动性风险 |
| `delisting_risk` | Boolean | 是否有退市风险 | 永久黑名单 |
| `trading_status` | String | 交易状态 | "NORMAL", "SUSPENDED", "DELISTED" |

**数据更新频率**: 每日盘前更新

---

## 5. 数据质量要求

### 5.1 数据完整性
- **缺失值处理**: 
  - 数值字段缺失时返回 `null`，不要返回 `0` 或 `-1`
  - 字符串字段缺失时返回 `null` 或空字符串 `""`
  
### 5.2 数据时效性
| 数据类型 | 延迟要求 | 说明 |
|---------|---------|------|
| 实时行情 | < 3 秒 | 交易时间内 |
| 盘口数据 | < 1 秒 | 用于流动性检查 |
| 日线数据 | 当日盘后 | T+0 更新 |
| 财务数据 | 3 个工作日 | 财报发布后 |

### 5.3 数据准确性
- **价格精度**: 保留 2 位小数
- **百分比精度**: 保留 2 位小数
- **成交额单位**: 统一使用"元"，不要使用"万元"或"亿元"
